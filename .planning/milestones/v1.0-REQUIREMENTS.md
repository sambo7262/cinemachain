## v1 Requirements — Final Status

### Data Layer (DATA)

- [x] **DATA-01**: System fetches movie metadata (poster, rating, year, genres) from TMDB API
  - **Outcome:** Delivered. TMDBClient service + PostgreSQL cache + nightly pre-warm job (Phase 2 + Phase 4).

- [x] **DATA-02**: System fetches actor metadata and filmography credits from TMDB API
  - **Outcome:** Delivered. `fetch_person` + `fetch_actor_credits` in TMDBClient. On-demand + background pre-fetch.

- [x] **DATA-03**: System caches TMDB data in PostgreSQL to respect rate limits
  - **Outcome:** Delivered. PostgreSQL cache with APScheduler nightly job pre-warming top 5,000 movies + 1,500 actors.

- [x] **DATA-04**: System cross-references Plex library to determine watched/unwatched state per movie
  - **Outcome:** Delivered via manual `Mark as Watched` (DATA-06). Plex sync was implemented but superseded.

- [~] **DATA-05**: System receives Plex webhook events to update watch state on playback completion
  - **Outcome:** DESCOPED. Plex webhook implemented in Phase 2 then removed in Phase 3 (plan 03-21). Plex integration proved unreliable in practice. Manual marking (DATA-06) is the production path. Move to v2 as optional integration if demand exists.

- [x] **DATA-06**: User can manually mark a movie as watched (fallback without Plex Pass)
  - **Outcome:** Delivered. `PATCH /movies/{id}/watched` + Mark as Watched button in GameSession UI.

### Movie Game (GAME)

- [x] **GAME-01**: User can start a Movie Game session by manually selecting any movie as the starting point
  - **Outcome:** Delivered. Movie search → session creation → Session Home Page hub. Verified on live NAS.

- [x] **GAME-02**: User sees an Eligible Actors panel: cast of current movie, excluding already-picked actors
  - **Outcome:** Delivered. `get_eligible_actors` endpoint with session-scoped exclusion. Verified on live NAS.

- [x] **GAME-03**: User selects an actor to view their Eligible Movies panel (unwatched filmography)
  - **Outcome:** Delivered. Async credits pre-fetch on actor selection resolves NAS timeout. Verified on live NAS.

- [x] **GAME-04**: Session tracks picked actors so no actor can repeat within a session
  - **Outcome:** Delivered. `picked_actor_ids` exclusion in eligibility query. Verified on live NAS (03-29).

- [x] **GAME-05**: User can sort Eligible Movies by genre, TMDB rating, or aggregated rating
  - **Outcome:** Delivered. Sortable columns (rating, runtime, year, MPAA) with null-stable two-pass sort.

- [x] **GAME-06**: User can toggle between unwatched-only or all movies with watched badges
  - **Outcome:** Delivered. `all_movies` toggle; watched movies shown with dimmed overlay.

- [x] **GAME-07**: Only unwatched movies are selectable in Game mode
  - **Outcome:** Delivered. Watched-gate on eligible endpoints (HTTP 423 if not watched); only unwatched selectable.

- [x] **GAME-08**: User requests a movie, triggering an automatic Radarr queue request
  - **Outcome:** Delivered. `request_movie` endpoint calls RadarrClient and returns status. Verified on live NAS.

### Query Mode (QUERY) — DESCOPED

- [~] **QUERY-01 through QUERY-07**: Query Mode (actor search, title search, genre browse, sort, filter, Radarr/Sonarr request)
  - **Outcome:** DESCOPED from v1. Phase 4 was reprioritized to caching/polish over Query Mode. All 7 QUERY requirements are moved to v2 as first-class features.

### Infrastructure (INFRA)

- [x] **INFRA-01**: App runs as Docker Compose stack (backend + PostgreSQL + frontend containers)
  - **Outcome:** Delivered. Three containers on `cinemachain-net` bridge network. Running on Synology NAS.

- [x] **INFRA-02**: All API keys and config managed via .env (TMDB, Radarr, operational knobs)
  - **Outcome:** Delivered. 13 variables in `.env.example` with descriptions. Real credentials replaced with placeholders. Secrets audit PASS.

- [x] **INFRA-03**: PostgreSQL data persists via Docker volumes across restarts
  - **Outcome:** Delivered. Volume bind-mount at `${DATA_DIR:-./data}/postgres`.

- [x] **INFRA-04**: App is accessible via Tailscale IP/hostname on the home network
  - **Outcome:** Delivered. Tailscale sidecar; app reachable at Tailscale IP.

### Production (PROD) — Added in Phase 7

- [x] **PROD-01**: All secrets loaded from .env; .env.example documents every variable with no real values
  - **Outcome:** Delivered. 13 documented variables, placeholder-only values, `.gitignore` covers `.env`.

- [x] **PROD-02**: Docker Compose yaml is clean and deployable by any user following the README
  - **Outcome:** Delivered. Generic `DATA_DIR` variable, `cinemachain-net` managed network, 148-line README.

- [x] **PROD-03**: Security review confirms no credentials in images, no open ports beyond required
  - **Outcome:** Delivered. No credential grep matches, Postgres on `127.0.0.1` only, no secret `ENV` in Dockerfiles, `.dockerignore` present.
