## Key Accomplishments

1. **Movie Game end-to-end** — Full 6-step actor-chain game loop verified on live NAS hardware (03-29). Session state, actor exclusion, watched gate, Radarr request, and chain history all working.

2. **Multi-session support** — Multiple named sessions can run concurrently. Home page session grid, archive/delete, CSV export/import with fuzzy actor validation and disambiguation dialog.

3. **Production-grade UI** — Poster-as-background on session views, sortable eligible movies table, sidebar filters (genre/runtime/MPAA/rating), chain history search, movie selection splash dialog with full metadata, RT scores via MDBList.

4. **Nightly TMDB cache** — APScheduler job pre-warms top 5,000 movies + 1,500 actors nightly, eliminating on-demand TMDB calls during gameplay. Configurable via `TMDB_CACHE_TOP_N` / `TMDB_CACHE_TOP_ACTORS`.

5. **In-app Settings + MDBList RT scores** — Fernet-encrypted settings storage in DB. RT Tomatometer scores from MDBList API displayed in eligible movies table, Now Playing tile, and movie splash dialog.

6. **Public deployment hardened** — All credentials replaced with placeholders in `.env.example`, `compose.yaml` genericized with `DATA_DIR` variable and compose-managed network, comprehensive README, Docker images publicly pullable without authentication.
