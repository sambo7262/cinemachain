Five fixes deployed to NAS via Docker Hub pull:

1. **Issue A fix** (plan 03.2-15): Cross-session contamination closed. `get_eligible_movies` and `request_movie` in game.py now filter `WatchEvent` by `session_id`. Movies watched in Session A remain eligible in Session B.

Combined view now returns DB-cached results immediately (no TMDB calls). Individual actor selection still lazy-fetches via the actor_id-scoped branch.

3. **Issue C fix** (plan 03.2-17): Session home after back navigation. The session `useQuery` in `GameSession.tsx` now has `refetchOnMount: "always"`, forcing a fresh fetch on every component mount. The Mark as Watched button will reliably render on remount.

4. **Issue D1 fix** (plan 03.2-18): Concession loading messages. A `useLoadingMessages` hook rotates through 8 movie-concession messages every 2 seconds. Wired into: eligible actors loading, eligible movies loading, CSV import validation.

5. **Issue D2 fix** (plan 03.2-18): CSV direct TMDB ID input. Each flagged row in the validation report now shows a "Or enter TMDB ID directly:" numeric input field in addition to suggestion buttons. Entering a valid ID resolves that row without re-uploading the CSV.

6. **Issue D3 fix** (plan 03.2-18): CSV last-row behavior. The imported session's `current_movie_tmdb_id` is the last CSV row's movie, with `current_movie_watched=False`. No Radarr calls are made during import.

**Test 1 — Cross-session contamination (Issue A)**

1. Find or create Session A with at least one watched movie (a WatchEvent exists for it).
2. Create a NEW Session B with a different starting movie.
3. Mark Session B's starting movie as watched. Click "Continue the chain."
4. Open the Eligible Movies tab (with or without an actor selected).

**PASS:** Movies that were watched in Session A appear in Session B's eligible list as selectable (not grayed out, `selectable: true`). They may show a "Watched" badge if `all_movies` toggle is on, but they must be selectable.
**FAIL:** Movies watched in Session A appear as grayed out / non-selectable in Session B.

**Test 2 — Combined-view speed (Issue B)**

1. Open an active session where the current movie has been marked as watched.
2. Click "Continue the chain" to enter tabs view.
3. WITHOUT selecting any actor, click the "Eligible Movies" tab.
4. Note the time from clicking the tab to seeing movie results appear.

**PASS:** A concession message appears within 1 second (e.g., "Popping the popcorn..."), then movies load within 5 seconds total. The message rotates every ~2 seconds while loading.
**FAIL:** Loading takes more than 10 seconds, or page hangs/times out, or no concession message appears.

**Also verify search still works:** Type a partial movie title in the search box while in combined view. Matching movies should filter immediately.

**Test 3 — Session home Mark as Watched after back navigation (Issue C)**

1. Start an active session. Mark the current movie as watched.
2. Click "Continue the chain" → enter tabs view.
3. Select an actor. Select a movie and confirm. The UI returns to Session Home showing the new movie.
4. Click "Back to Sessions" (or use browser/NavBar to navigate away from the session).
5. Navigate back into the same session from the sessions list.

**PASS:** The Session Home Page shows the Mark as Watched button for the new movie within 2 seconds of loading. The button label reads "Mark as Watched" and is enabled.
**FAIL:** Session Home shows no Mark as Watched button and no other action for the new current movie.

**Test 4 — Concession loading messages (Issue D1)**

Verify in three places:

4a. Eligible Actors loading: open a session, watch the current movie, click "Continue the chain" → open Eligible Actors tab. If actors take >1s to load, a rotating concession message appears.

4c. CSV import loading: go to Lobby → Start new session → Import Chain tab → upload a CSV → click Import Chain. While validation runs, a rotating concession message appears (e.g., "Popping the popcorn...").

**PASS:** At least one concession message is visible in each of the three loading scenarios. Messages rotate approximately every 2 seconds.
**FAIL:** Generic spinner or no message during loading.

**Test 5 — CSV direct TMDB ID input (Issue D2)**
