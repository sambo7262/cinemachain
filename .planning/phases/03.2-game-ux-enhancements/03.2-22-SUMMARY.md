## Accomplishments

- Fixed Mark as Watched button not appearing after Back to Sessions + re-enter by adding `staleTime: 0` to the session query — stale cache holding `current_movie_watched: true` was preventing the button from rendering even though the server had fresh `false` state
- Suppressed "No eligible movies found" empty-state text while `eligibleMoviesFetching` is true, eliminating false-empty flash during loading
- Slowed concession message rotation from 2000ms to 3000ms for a less frenetic loading experience
- Confirmed ChainHistory (Watch History) already correctly hidden when `session.steps.length === 0` — no change required

## Files Created/Modified

- `frontend/src/hooks/useLoadingMessages.ts` - Changed setInterval from 2000ms to 3000ms

## Decisions Made

- `staleTime: 0` is the correct fix: React Query's default staleTime means the cached value (potentially `current_movie_watched: true` from a previous session state) is served as fresh. With `staleTime: 0`, the cached value is immediately treated as stale on mount, so `refetchOnMount: "always"` fires and the fresh server response replaces the cached value before the button condition evaluates.
- `ChainHistory` confirmed already guarded — no Watch History tab trigger exists in the TabsList; the section is conditionally rendered directly on the page.
