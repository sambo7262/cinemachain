Root cause confirmed from code:

In `handleMovieConfirm` (GameSession.tsx line 158-168), after `api.requestMovie` succeeds, the code calls:
```ts
queryClient.invalidateQueries({ queryKey: ["session", sid] })
```
This marks the cache stale and schedules a background refetch. If the user navigates away before that refetch completes, the cache retains the `continue_chain` response (injected earlier via `setQueryData` in `handleContinue`) where `current_movie_watched: true`. On return navigation, React Query serves this stale value — `isWatched` evaluates to `true` — and the Mark as Watched button at line 362 is hidden because its condition is `session.status === "active" && !isWatched`.

The `request_movie` response already includes a fully populated session object (`requestResult.session: GameSessionDTO`) with `current_movie_watched: false`. The same synchronous-setQueryData pattern is already used correctly in:
- `markWatchedMutation.onSuccess` (line 187): `queryClient.setQueryData(["session", sid], updatedSession)`
- `handleContinue` (line 198): `queryClient.setQueryData(["session", sid], updatedSession)`

**Fix:** In `handleMovieConfirm`, add `queryClient.setQueryData(["session", sid], requestResult.session)` immediately after `api.requestMovie` returns and BEFORE `invalidateQueries`. This writes `current_movie_watched: false` into the cache synchronously, so even if the user navigates away mid-refetch, the cached data is correct.

The call to `invalidateQueries` can remain — it ensures the session is eventually re-fetched fresh from the server, which is still desirable.

Find the block at approximately line 158-169:
```ts
      const requestResult = await api.requestMovie(sid, {
        movie_tmdb_id: movie.tmdb_id,
        movie_title: movie.title,
      })
      if (requestResult?.status === "already_in_radarr") {
        setRadarrStatus("Already in your library — waiting for watched event.")
      } else if (requestResult?.status === "queued") {
        setRadarrStatus("Added to Radarr queue.")
      }
      setView("home")
      queryClient.invalidateQueries({ queryKey: ["session", sid] })
      queryClient.invalidateQueries({ queryKey: ["eligibleActors", sid] })
```

After the `setRadarrStatus` / `setView("home")` block and BEFORE the `invalidateQueries` calls, insert:
```ts
      queryClient.setQueryData(["session", sid], requestResult.session)
```

No other changes. Do not alter the invalidateQueries calls, the error handling, or any other logic.
