## Accomplishments
- Raised httpx timeout to 60s connect / 90s read — eliminates ConnectTimeout on NAS for large CSVs
- Implemented _title_confidence scoring (high/medium/low/none) and refactored _resolve_movie_tmdb_id to return top-3 suggestions with confidence tuple
- Rewrote import_csv_session with two-pass validate-first logic: high/medium confidence rows auto-resolve, low/none rows return 200+validation_required without creating a session
- Added re-submission flow: body.overrides maps row indexes to confirmed tmdb_ids, skipping TMDB on second pass
- Added frontend suggestion picker UI with loading message during validation, per-row suggestion buttons, and Confirm and Import re-submit button

## Decisions Made
- httpx.Timeout named-parameter form used to set connect and read independently; prevents ConnectTimeout on NAS for 136-row CSVs
- medium confidence auto-accepts (contains-match is likely user typo); only low and none trigger the suggestion picker to avoid excessive false-positive flags
- import re moved to module-level as _re to avoid repeated import inside helper function
- response_model removed from import-csv decorator; JSONResponse returned for validation_required, dict returned for session creation
- CsvValidationResponse status is HTTP 200 not 201 so frontend mutation onSuccess handler receives it (not onError)
