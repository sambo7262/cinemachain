## Decisions Made
- `_ensure_movie_cast_in_db` uses `pg_insert on_conflict_do_update` for Actor rows (to keep name/profile_path current) and `on_conflict_do_nothing` for Credit rows (idempotent, avoids duplicates)
- Helper degrades gracefully on TMDB failure (bare `except Exception: return`) — same pattern as existing helpers
- `db.flush()` used inside loop per-actor before PK resolve, single `db.commit()` at end — matches batch insert best practice
