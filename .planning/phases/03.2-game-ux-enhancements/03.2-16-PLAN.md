<objective>
Remove sequential `_ensure_actor_credits_in_db` calls from the combined-view path in `get_eligible_movies`.

Purpose: Lines 1194–1196 in game.py loop over every eligible actor and await `_ensure_actor_credits_in_db` sequentially. With 20+ eligible actors this causes 20+ sequential TMDB calls (each up to 2s on NAS) before any response is sent — this is the root cause of the 30-60s combined-view load time. The combined-view path should use only credits already in the DB; individual actor enrichment happens lazily when the user selects a specific actor (the actor_id-scoped branch already handles this at line 1149).

Search is already wired client-side in `GameSession.tsx` line 139: `debouncedSearch` filters `allEligibleMovies`. This works correctly once the combined-view response returns promptly. No frontend changes needed.

Output: `backend/app/routers/game.py` with `_ensure_actor_credits_in_db` loop removed from the combined-view (else) branch.
</objective>
