<objective>
Force the session query to always refetch fresh data on component mount in `GameSession.tsx`.

Purpose: When the user navigates: pick movie → confirm → Back to Sessions → re-enter session, the component remounts. React Query's default `refetchOnMount: true` refetches data that is considered stale (staleTime=0 by default), but in practice the 5-second polling interval means there can be a brief window where React Query serves the cached value before the refetch completes. By setting `refetchOnMount: "always"`, React Query initiates a fetch immediately on every mount regardless of cache freshness, ensuring the `Mark as Watched` button renders as soon as the fresh response arrives rather than potentially showing stale state.

The `Mark as Watched` button renders at line 377 in `GameSession.tsx` when `session.status === "active" && !isWatched`. After `request_movie`, the session has `current_movie_watched: false`, so `isWatched` is false. A stale cache entry with `current_movie_watched: true` (from the prior `awaiting_continue` state) would suppress the button.

Output: `frontend/src/pages/GameSession.tsx` session query with `refetchOnMount: "always"`.
</objective>
