<objective>
Fetch full TMDB movie details (genres + runtime) on-demand for movie stubs that are missing genre data, enabling the runtime filter, genre filter, and watched_runtime_minutes counter to work correctly.

Purpose: Three of the five gaps (runtime filter no effect, genre filter empty, watched_runtime_minutes=0) share a single root cause — `_ensure_actor_credits_in_db` inserts movie stubs from `/person/{id}/movie_credits` with `genres=None` and `runtime=None` because that endpoint only returns `genre_ids` (integers) and no runtime. Full movie details require a separate `GET /movie/{id}` call that returns `genres` (objects with names) and `runtime` (minutes). This plan adds an `_ensure_movie_details_in_db` helper that fetches full movie details on-demand for movies with `genres IS NULL`, and calls it during the `eligible-movies` endpoint for the current page.

Output: `backend/app/routers/game.py` with `_ensure_movie_details_in_db` helper, call in `get_eligible_movies`, and updated `_enrich_steps_runtime` path that benefits from populated runtime data.
</objective>

Behavior:
1. Query `Movie.genres IS NULL` for the given `tmdb_ids` to find movies that need a full details fetch:

   rows = await db.execute(
       select(Movie.tmdb_id).where(
           Movie.tmdb_id.in_(tmdb_ids),

       )
   )
   needs_fetch = [row[0] for row in rows.all()]

2. For each `tmdb_id` in `needs_fetch`, call `tmdb.fetch_movie(tmdb_id)` (existing method — hits `/movie/{id}?append_to_response=credits`, returns `genres` list and `runtime` int). Swallow exceptions per movie (degrade gracefully).

   ```python
   import json
   genres_json = json.dumps([g["name"] for g in data.get("genres", [])])
   runtime_val = data.get("runtime")
   await db.execute(
       sa.update(Movie).where(Movie.tmdb_id == tmdb_id).values(
           genres=genres_json,
           runtime=runtime_val,
       )
   )
   ```
4. `await db.commit()` after each batch (or per-movie inside try/except — per-movie is safer for partial failures).
