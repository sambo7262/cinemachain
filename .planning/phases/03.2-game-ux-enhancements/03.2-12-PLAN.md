  <what-built>
Five fixes deployed to NAS via Docker Hub pull:

1. **Regression 1 fix** (plan 03.2-10): `get_eligible_actors` on-demand fallback now uses a freshly constructed SELECT statement after inserts complete, preventing stale-transaction cache from returning zero actors.

2. **Gap 3 fix** (plan 03.2-11): Eligible Movies tab now fetches ALL eligible movies immediately when opened with no actor selected (backend combined-view was already correct; frontend now enables the query unconditionally).

3. **Regression 2 fix** (plan 03.2-11): queryKey for eligible-movies now uses `selectedActor?.tmdb_id ?? null` — React Query treats no-actor and actor-present as distinct cache entries, preventing stale actor data from flashing.

4. **Mark as Watched nav bug fix** (plan 03.2-13): `handleMovieConfirm` now calls `queryClient.setQueryData(["session", sid], requestResult.session)` immediately after `request_movie` succeeds, writing `current_movie_watched: false` into the React Query cache synchronously. Previously, navigating away before the background refetch completed left the cache with `current_movie_watched: true` from the prior `continue_chain` response, hiding the Mark as Watched button on return.

5. **CSV validate-first import** (plan 03.2-14): httpx connect timeout raised to 60s. `import_csv_session` now runs a full validation pass, auto-accepts high/medium confidence matches, and returns `{ status: "validation_required", unresolved: [...] }` for ambiguous rows instead of 500-ing. Re-submission with confirmed `overrides` creates the session. Frontend shows a loading message during validation, suggestion pickers per flagged row, and a "Confirm and Import" button once all rows are resolved.
  </what-built>
