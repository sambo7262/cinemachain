**Fix 1 — Diagnose and fix Mark as Watched button not appearing after remount.**

Read the session query at lines 54–61 and the isWatched derivation at line 64. Current state:
```typescript
const { data: session } = useQuery({
  queryKey: ["session", sid],
  queryFn: () => api.getSession(sid),
  refetchInterval: (query) =>
    query.state.data?.status === "awaiting_continue" ? false : 5000,
  refetchOnMount: "always",
  enabled: !!sid,
})
const isWatched: boolean = session?.current_movie_watched ?? false
```

The render condition at lines 370–379:
```tsx
{session.status === "active" && !isWatched && (
  <Button onClick={() => markWatchedMutation.mutate()} ...>
    Mark as Watched
  </Button>
)}
```

The problem: On first render after remount, `session` is `undefined` (React Query hasn't returned data yet), so `isWatched` defaults to `false` and `session.status` is undefined — the render condition fails because `session` is falsy. The parent `{view === "home" && session && (...)}` at line 328 should gate this, but if the stale session cached before navigation has `current_movie_watched: true` (from the previous awaiting_continue state), then after remount the cached value serves `isWatched = true` and hides the button — even though the fresh refetch would return `current_movie_watched: false`.

**Root fix:** Force the session query to discard stale cache on remount by adding `staleTime: 0` to ensure the refetch always overrides cached data before rendering the button.

Update the session query options:
```typescript
const { data: session, isLoading: sessionLoading } = useQuery({
  queryKey: ["session", sid],
  queryFn: () => api.getSession(sid),
  refetchInterval: (query) =>
    query.state.data?.status === "awaiting_continue" ? false : 5000,
  refetchOnMount: "always",
  staleTime: 0,
  enabled: !!sid,
})
```

Adding `staleTime: 0` means every mount triggers an immediate refetch and the cached value is considered stale — the fresh server response replaces it before stable rendering occurs.

Also add `sessionLoading` to the destructuring (used in Fix 2 below).
