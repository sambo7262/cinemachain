## New Regressions Identified

### Regression 1 — Actor eligibility after _ensure_movie_details_in_db
**Chain tested:** The Martian → Matt Damon → Good Will Hunting → Minnie Driver → South Park: Bigger Longer Uncut
**Symptom:** No eligible actors returned for South Park: Bigger Longer Uncut.
**Likely cause:** `_ensure_movie_details_in_db` introduced a bug in actor eligibility logic — possibly overwriting `credits` data or altering `actor_tmdb_id` values in a way that breaks the `get_eligible_actors` exclusion query.
**Impact:** Core game loop broken for movies fetched via `_ensure_movie_details_in_db`. Blocks gameplay on any movie that was a stub before eligibility check.

### Regression 2 — Stale movie list on actor change
**Symptom:** After advancing the chain and selecting a different actor, the previous actor's movie list briefly persists before the new list appears.
**Cause:** React Query cache invalidation is not firing on actor state change; the `eligibleMovies` query key is likely not including the selected actor ID, or the key is not being invalidated when the actor changes.
**Impact:** UX confusion — players see incorrect movie list momentarily. Not a game-breaking regression but a noticeable UX defect.
