<objective>
Fix _prefetch_credits_background to use _ensure_movie_cast_in_db (1 TMDB call) instead of calling _ensure_actor_credits_in_db per cast member (up to 20 calls). Eliminates TMDB rate-limit exhaustion for movies with large-filmography cast members (e.g. South Park's Trey Parker and Matt Stone).

Purpose: The current background pre-fetch calls /person/{id}/movie_credits once per actor — up to 20 separate TMDB calls. For cast members with hundreds of credits (South Park), this exhausts the rate limit before session creation completes. The on-demand fallback then also fails, returning only already-cached actors.

Output: game.py with a 4-line _prefetch_credits_background that delegates to _ensure_movie_cast_in_db.
</objective>

```python
async def _prefetch_credits_background(
    movie_tmdb_id: int,
    tmdb: TMDBClient,
) -> None:
    """Background task: populate Credits for the starting movie's cast in a single TMDB call.

    Uses _ensure_movie_cast_in_db (1 call to /movie/{id}/credits) instead of
    _ensure_actor_credits_in_db per actor (20 calls to /person/{id}/movie_credits).
    This eliminates TMDB rate-limit exhaustion for movies with large-filmography cast members.
    Actor filmographies are fetched on-demand when the user selects an actor.
    """
    try:
        async with _bg_session_factory() as db:
            await _ensure_movie_cast_in_db(movie_tmdb_id, tmdb, db)
    except Exception:
        pass
```

The new implementation delegates entirely to _ensure_movie_cast_in_db which already calls /movie/{id}/credits once and batch-upserts Actors and Credits. _ensure_movie_cast_in_db already has its own try/except for the TMDB call so the outer try/except here only guards the _bg_session_factory context manager.
