<objective>
Implement Gap 3 (no-actor eligible movies) and fix Regression 2 (stale movie list on actor change).

Gap 3: When the user opens the Eligible Movies tab without selecting an actor, the frontend currently shows an empty state. The backend already has a combined-view branch that returns all eligible movies when `actor_id` is omitted — the fix is purely frontend: enable the query unconditionally (no actor required) and pass `actor_id: undefined` when no actor is selected.

Regression 2: The queryKey for eligible-movies already includes `selectedActor?.tmdb_id`, but switching actors briefly shows stale data. Fix: use `selectedActor?.tmdb_id ?? null` (not `undefined`) in the queryKey so React Query treats missing-actor and actor-present as distinct cache keys, and add a loading spinner that shows after 1 second when `isFetching` is true.

Purpose: UX-03 requires the Eligible Movies tab to be immediately useful without actor selection. The backend combined-view already handles this; the frontend just needs to stop gating the fetch on actor selection.
