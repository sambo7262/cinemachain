## Decisions Made
- eligibleActorsData holds the full API result; eligibleActors (used for the eligible table) is filtered client-side to `is_eligible !== false`. The ineligible section reads directly from eligibleActorsData to avoid double-fetching.
- filteredMovies is computed from allEligibleMovies (the current paginated page). Load More pagination still drives new page fetches; filters apply within each loaded page. This matches the plan's note to keep pagination operating on the full list.
- parseGenres utility placed as a module-level const in GameSession.tsx (not exported) — avoids coupling without requiring a new shared utility file.
- SessionCounters rendered inside the header's flex-col div alongside session name and Now Playing line, satisfying the UI-SPEC placement "between session name and tab panel."
