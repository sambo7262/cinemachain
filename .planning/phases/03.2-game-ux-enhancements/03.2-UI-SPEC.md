## Spacing Scale

Standard 8-point scale. All values are multiples of 4.

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge padding, inline padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, table cell padding (`px-3 py-2` = 12/8px — existing pattern) |
| md | 16px | `p-4` / `gap-4` | Default element spacing, card padding |
| lg | 24px | `p-6` / `gap-6` | Section padding, sidebar padding |
| xl | 32px | `p-8` / `gap-8` | Layout gaps |
| 2xl | 48px | `py-12` / `gap-12` | Major section breaks |
| 3xl | 64px | `py-16` / `gap-16` | Page-level spacing |

Exceptions:
- Movie thumbnail in ChainHistory: exactly `w-10 h-14` (40×56px) — fixed ratio, not a spacing token
- Actor thumbnail in ChainHistory: exactly `w-7 h-7` (28×28px) — fixed size circle, not a spacing token
- Actor thumbnail in Eligible Actors tab: `w-8 h-8` (32×32px) — existing pattern from `GameSession.tsx`
- Touch targets for toggle/filter buttons: minimum 44px height

## Color

### Light Mode

| Role | CSS Variable | Resolved Value | Tailwind Token |
|------|-------------|----------------|----------------|
| Dominant (60%) — background | `--background` | `hsl(0, 0%, 100%)` = white | `bg-background` |
| Secondary (30%) — cards, sidebar, table header | `--card` / `--muted` | `hsl(240, 4.8%, 95.9%)` = near-white zinc | `bg-card` / `bg-muted` |
| Accent (10%) | `--primary` | `hsl(240, 5.9%, 10%)` = near-black zinc | `bg-primary text-primary-foreground` |
| Destructive | `--destructive` | `hsl(0, 84.2%, 60.2%)` = red | `bg-destructive text-destructive-foreground` |

### Dark Mode

| Role | CSS Variable | Resolved Value |
|------|-------------|----------------|
| Dominant (60%) | `--background` | `hsl(240, 10%, 3.9%)` = near-black zinc |
| Secondary (30%) | `--card` / `--muted` | `hsl(240, 3.7%, 15.9%)` = dark zinc |
| Accent (10%) | `--primary` | `hsl(0, 0%, 98%)` = near-white |
| Destructive | `--destructive` | `hsl(0, 62.8%, 30.6%)` = dark red |

Accent (`bg-primary`) is reserved for:
- "Select" / primary CTA buttons in eligible actor and movie rows
- Active tab indicator
- "Show ineligible actors" toggle when active state

Accent is NOT used for: filter chips, badges, muted labels, table headers, or any informational element.

Semantic colors:
- `bg-muted / text-muted-foreground`: filter sidebar header, table header row (`bg-muted/50`), actor row background (`bg-muted/20`), ineligible actor section background
- `text-muted-foreground`: all caption text, already-picked actor names, "Already picked" badge text, counter labels
- `border-border`: table dividers, sidebar border, filter category separators

## Interaction Contracts

### Filter Sidebar

- Rendered as a fixed-width panel (`w-56` = 224px) to the left of the movie grid/list
- On narrow screens (`sm` breakpoint and below): sidebar collapses into a "Filters" button that opens a drawer or sheet — implementation at executor's discretion, but must not obstruct movie list
- Each filter category (`Runtime`, `Genre`, `MPAA Rating`) uses a `Collapsible` with `aria-expanded`
- All categories start collapsed (`defaultOpen={false}`)
- Expanding one category does NOT collapse others
- Filter state is local component state — no URL params, no backend calls

#### Runtime Slider

- `Slider` component, two thumbs (min/max)
- Range: 0 to 300 minutes (5 hours — covers all realistic films)
- Step increment: 15 minutes
- Default: [0, 300] = "no filter applied"
- Label format: `{min}m – {max}m` (e.g. "90m – 150m"); display `{n}h {m}m` if >= 60 minutes (e.g. "1h 30m – 2h 30m")
- When range is [0, 300], display "Any runtime" instead of the full range
- Movies with `runtime == null` always pass the runtime filter (do not exclude unknown-runtime movies)

#### Genre Multi-Select

- `Checkbox` per genre, one per line
- Genre options derived from loaded movie list: `[...new Set(eligibleMovies.flatMap(m => parseGenres(m.genres)))].sort()` — alphabetical order
- Zero checkboxes checked = "no filter" (all movies pass)
- Checked genres use OR logic: movie passes if it matches ANY checked genre

#### MPAA Rating Checkboxes

- Fixed option set: `G`, `PG`, `PG-13`, `R`, `NR` (in this order)
- `NR` covers both `mpaa_rating === null` and `mpaa_rating === ""`
- Zero checked = "no filter"
- Checked ratings use OR logic

#### Active Filter Indicator

- When any filter is non-default, show a small count badge on the sidebar header: e.g. "Filters (2)"
- Badge uses `bg-primary text-primary-foreground` to signal active state

### Movie Title Search

- `Input` component placed above the movie list, full width of the movie panel area
- Placeholder: "Search movies..."
- Debounce: 200ms (via `useEffect` + `setTimeout` — no library)
- Filters the `eligibleMovies` array in place — not a new API call
- Search and sidebar filters apply simultaneously (AND relationship between search and sidebar filters)
- When search is active and produces zero results: show empty state (see Copywriting section)

### Ineligible Actor Toggle

- `Button` variant `outline` placed above the eligible actors list
- Label when inactive: "Show already-picked actors"
- Label when active: "Hide already-picked actors"
- Toggle state is local component state (`showIneligible: boolean`, default `false`)
- `showIneligible` must be included in the `queryKey` for eligible actors (prevents stale cache serving wrong data)

#### Ineligible Actor Section

- Rendered below all eligible actors, separated by a `Separator` with label "Already picked"
- Each ineligible actor row: same structure as eligible actor row
- Visual treatment: `opacity-50` on the entire row container; avatar uses `bg-muted` placeholder regardless of `profile_path` availability
- "Already picked" badge: `<Badge variant="secondary">Already picked</Badge>` — placed after actor name
- Ineligible actors are NOT selectable — no click handler, `cursor-default` on the row

### Chain History Thumbnails

- Movie thumbnail column width: `w-10` (40px); thumbnail height: `h-14` (56px) — 5:7 poster ratio, `object-cover`, `rounded`
- When `poster_path` is non-null: `<img src={\`https://image.tmdb.org/t/p/w92${poster_path}\`} ... />`
- When `poster_path` is null: existing `bg-muted` placeholder div (film icon SVG, already in `ChainHistory.tsx`)
- Actor thumbnail: `w-7 h-7` (28×28px) circle
- When `profile_path` is non-null: `<img src={\`https://image.tmdb.org/t/p/w45${profile_path}\`} ... className="w-7 h-7 rounded-full object-cover" />`
- When `profile_path` is null: existing initials placeholder div (`bg-muted rounded-full`, 2-letter initials) — already in `ChainHistory.tsx`
- TMDB image base URL: `https://image.tmdb.org/t/p/` — no trailing slash before size token

### Session Counters

- Placement: top of session page, between session name and the tab panel — same area as Radarr status message
- Layout: two counters side by side, separated by a vertical `Separator` or `|` character
- Counter 1 label: "Watched" — value: integer (e.g. "3")
- Counter 2 label: "Runtime" — value: `Xh Ym` format
  - If `watched_runtime_minutes < 60`: display as `{n}m` (no hours component)
  - If `watched_runtime_minutes >= 60`: display as `{h}h {m}m` (e.g. "2h 14m")
  - If `watched_runtime_minutes === 0` and `watched_count === 0`: display "—" for runtime (not "0h 0m")
- Counter values: `text-sm font-semibold text-foreground`
- Counter labels: `text-xs text-muted-foreground uppercase tracking-wide`

## State and Visibility Contracts

### Filter Panel States

| State | Trigger | Visual |
|-------|---------|--------|
| Filter collapsed (default) | On mount | Category header visible, content hidden, chevron pointing right |
| Filter expanded | User clicks category header | Content visible, chevron pointing down |
| No filter active | All inputs at defaults | Sidebar header: "Filters" (no badge) |
| Filter active | Any input non-default | Sidebar header: "Filters (N)" with primary badge |

### Ineligible Toggle States

| State | Visual |
|-------|--------|
| `showIneligible === false` (default) | Button label: "Show already-picked actors"; no ineligible section rendered |
| `showIneligible === true` | Button label: "Hide already-picked actors"; ineligible section rendered below eligible list |
| API loading ineligible data | Button disabled with spinner while query refetches |

### Thumbnail States

| State | Movie thumbnail | Actor thumbnail |
|-------|----------------|----------------|
| `poster_path` / `profile_path` available | TMDB image rendered | TMDB image rendered |
| `poster_path` / `profile_path` null | `bg-muted` placeholder with film SVG icon (movie) or initials (actor) | Same |
| Image load error | Fall back to placeholder via `onError` handler | Same |

### Session Counter States

| State | Display |
|-------|---------|
| No movies watched yet | Watched: 0, Runtime: — |
| Movies watched, some missing runtime | Runtime sum includes only movies with non-null `Movie.runtime` |
| Movies watched, all have runtime | Full sum displayed |

## Accessibility Contract

- `Collapsible` uses Radix primitive — provides `aria-expanded` and keyboard (Enter/Space) support automatically
- `Checkbox` uses Radix primitive — provides `aria-checked`, keyboard, and `role="checkbox"` automatically
- `Slider` uses Radix primitive — provides `aria-valuemin`, `aria-valuemax`, `aria-valuenow`, keyboard (arrow keys) automatically
- Ineligible actor rows: `aria-disabled="true"` on the row, remove any `onClick` handlers
- Search input: `aria-label="Search eligible movies"` (in addition to placeholder)
- Session counters: wrap in `<dl>` with `<dt>` labels and `<dd>` values for semantic markup
- Thumbnail images: `alt={movie.title}` / `alt={actor.name}`; placeholder divs: `aria-hidden="true"`
