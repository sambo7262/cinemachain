# Phase 03.2 Plan 23: WatchEvent Batch Insert for CSV Import Summary

**CSV import now batch-inserts WatchEvent records (source="csv_import") for all prior chain movies so session watched_count and watched_runtime_minutes are non-zero immediately after import.**

## Accomplishments
- `import_csv_session` now inserts WatchEvent rows for every prior movie step (all movies except the current in-progress last movie) using `pg_insert(WatchEvent).on_conflict_do_nothing(index_elements=["tmdb_id"])`
- `source="csv_import"` tag distinguishes these events from manual marks (`source="manual"`)
- Session counters (watched_count, watched_runtime_minutes) derived by `_build_session_response` from WatchEvent data now correctly reflect the full imported chain history rather than zeros
- A 10-movie CSV import now immediately shows 9 movies watched and accumulated runtime

## Files Created/Modified

## Decisions Made
- `prior_movie_ids` uses a set comprehension over `steps_data` (the in-memory list built during CSV parsing) filtering for `actor_tmdb_id is None` (movie-pick steps only) and `movie_tmdb_id != last_movie_id` (exclude current in-progress movie). This is clean and avoids a second DB query.
- WatchEvent inserts are committed in a second `db.commit()` call so the rows exist before `_build_session_response` reads them to compute `watched_count`.
- `on_conflict_do_nothing(index_elements=["tmdb_id"])` makes the insert idempotent — safe to re-run on duplicate import attempts.
