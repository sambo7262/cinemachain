# Phase 03.2 Plan 28: Fix _prefetch_credits_background TMDB Rate-Limit Exhaustion Summary

**One-liner:** Replaced per-actor _ensure_actor_credits_in_db loop in background prefetch with single _ensure_movie_cast_in_db call, reducing TMDB calls from 1+20 to 1 per session creation.

## What Was Done

Task 1 replaced `_prefetch_credits_background` (lines 468-488 of game.py) with a 4-line implementation that delegates entirely to `_ensure_movie_cast_in_db`. The old body fetched `/movie/{id}/credits` itself, then looped over up to 20 cast members calling `_ensure_actor_credits_in_db` for each — which calls `/person/{id}/movie_credits` per actor. For South Park cast members like Trey Parker and Matt Stone (hundreds of credits), this exhausted the TMDB rate limit before session creation completed, leaving the on-demand fallback with nothing to return.

The new implementation:
```python
try:
    async with _bg_session_factory() as db:
        await _ensure_movie_cast_in_db(movie_tmdb_id, tmdb, db)
except Exception:
    pass
```

`_ensure_movie_cast_in_db` already makes exactly 1 TMDB call (`/movie/{id}/credits`), batch-upserts Actors and Credits, and has its own try/except — making the outer guard purely for the session factory context manager.
