## Accomplishments

- Added `_ensure_movie_details_in_db` helper that queries Movie rows with `genres IS NULL` for the current page's tmdb_ids, calls `tmdb.fetch_movie(id)` per missing row, stores `genres` as a JSON string (list of genre names) and `runtime` as integer minutes
- Wired the helper into `get_eligible_movies` before `movies = list(movies_map.values())` so all downstream sort/filter operations have populated values
- Added movies_map refresh block after the helper call so in-memory dict reflects the newly committed DB values
- Added `import json` to game.py (was absent)

## Decisions Made

- `_ensure_movie_details_in_db` fetches per-movie inside individual try/except blocks so a single TMDB timeout or HTTP error degrades gracefully without blocking remaining movies
- Refresh query after the helper re-reads `Movie.tmdb_id, Movie.genres, Movie.runtime` for all keys in movies_map, ensuring both newly-fetched and pre-existing rows get their current values reflected
- Helper placed between `_ensure_actor_credits_in_db` and `_prefetch_credits_background` to maintain logical grouping of on-demand fetch helpers
