<objective>
After CSV import, create WatchEvent records for all imported movie steps except the last one (which is the in-progress current movie). This makes session counters (movies watched, total runtime) reflect the full imported chain history immediately after import.

Purpose: Currently, import_csv_session creates steps but no WatchEvent records. The session counters show 0 watched / 0 runtime because _build_session_response derives watched_count and watched_runtime_minutes from WatchEvent data. A user importing a 10-movie chain expects to see 9 movies watched and the accumulated runtime immediately — not zeros. The fix: after committing all steps, batch-insert WatchEvent rows for every movie step where actor_tmdb_id IS NOT NULL (i.e., prior movies in the chain, not the current in-progress movie).

Output: import_csv_session inserts WatchEvent records for prior steps; session counters correct after import.
</objective>

    # Create WatchEvent records for all prior movie steps (not the last in-progress movie).
    # This ensures session counters (watched_count, watched_runtime_minutes) reflect the
    # full imported chain history immediately — not zeros.
    # Identify "prior" movies: all movie steps with a subsequent actor step in the chain.
    # In the steps_data list: a movie step is "prior" if it is NOT the last movie
    # (i.e., not the current_movie_tmdb_id).
    prior_movie_ids = list({
        s["movie_tmdb_id"]
        for s in steps_data
        if s["actor_tmdb_id"] is None  # movie-pick steps only
        and s["movie_tmdb_id"] != last_movie_id  # exclude current in-progress movie
    })
    if prior_movie_ids:
        now = _datetime.utcnow()
        for mid in prior_movie_ids:
            we_stmt = pg_insert(WatchEvent).values(
                tmdb_id=mid,
                source="csv_import",
                watched_at=now,
            ).on_conflict_do_nothing()
            await db.execute(we_stmt)
        await db.commit()
