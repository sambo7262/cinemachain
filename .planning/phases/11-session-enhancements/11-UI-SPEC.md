## Typography

Pre-populated from existing `GameSession.tsx` patterns. Source: code inspection of table and filter bar.

Focal point: The movie title column is the primary visual anchor; poster thumbnail is the secondary orientation cue.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 |
| Label / table header | 14px (`text-sm`) | 400 regular | 1.5 |
| Section heading | 16px (`text-base`) | 600 semibold (`font-semibold`) | 1.2 |
| Display / now-playing title | 20px (`text-xl`) | 600 semibold (`font-semibold`) | 1.2 |

Notes:
- Muted text uses `text-muted-foreground` at 14px for supporting copy (table metadata, timestamps).
- Icon size in poster overlay: `w-3.5 h-3.5` (14px) — matches body text size.
- Filter toggle button labels use `text-sm` (14px) matching the existing "All Movies / Unwatched Only" button.
- Table headers previously using `font-medium` (500) are reclassified to 400 regular to stay within the 2-weight limit.
- Display title previously using `font-bold` (700) is reclassified to `font-semibold` (600).

## Color

The app renders exclusively in dark mode (`<html class="dark">` in `index.html`). All color references below are dark-mode values from `src/index.css`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(240 10% 3.9%)` — `bg-background` | Page background, table body |
| Secondary (30%) | `hsl(240 3.7% 15.9%)` — `bg-card` / `bg-secondary` | Filter sidebar, card panels, table header row |
| Accent (10%) | `hsl(240 3.7% 15.9%)` — shadcn `accent` | Hover state on table rows (`hover:bg-accent/50`) |
| Destructive | `hsl(0 62.8% 30.6%)` — shadcn `destructive` | "Clear Shortlist" button destructive variant only |

Accent reserved for: table row hover state only. All other highlighted states use semantic Tailwind colors (amber for saved, blue for shortlisted) that operate outside the shadcn accent system.

### Phase-specific semantic colors (source: CONTEXT.md D-08)

| State | Class | Description |
|-------|-------|-------------|
| Saved | `bg-amber-500/10` | 10% amber overlay on `<tr>` when `movie.saved === true` |
| Shortlisted | `bg-blue-500/10` | 10% blue overlay on `<tr>` when `movie.shortlisted === true` |
| Both saved + shortlisted | Apply both `bg-amber-500/10` and `bg-blue-500/10` via `cn()` | Tailwind renders additive alpha; warm-cool blend is intentional; both classes must be literal strings (no dynamic interpolation — Tailwind JIT purge requirement) |

### Icon color states

| Icon | Unsaved/inactive state | Saved/active state |
|------|------------------------|-------------------|
| `Star` (save) | `text-white` (outline, no fill) | `fill-amber-400 text-amber-400` |
| `ListCheck` (shortlist) | `text-white` (outline, no fill) | `fill-blue-400 text-blue-400` |

Icon background: `bg-black/40` at rest, `hover:bg-black/60` on hover (semi-transparent dark pill behind icon on poster).

## Copywriting Contract

Source: CONTEXT.md + RESEARCH.md Pitfall 3 guidance for filter-aware empty states.

| Element | Copy |
|---------|------|
| Primary CTA (save action) | "Save" (icon-only in practice; accessible aria-label: "Save movie") |
| Primary CTA (shortlist action) | "Shortlist" (icon-only in practice; accessible aria-label: "Add to shortlist") |
| Save toggle — saved state aria | "Remove from saved" |
| Shortlist toggle — shortlisted state aria | "Remove from shortlist" |
| Filter button label — saved active | "Saved ★" |
| Filter button label — shortlist active | "Shortlist ✓" |
| Clear shortlist button | "Clear Shortlist" (with Trash2 icon) |
| Empty state — Saved filter active | Heading: "No saved movies" · Body: "Save movies using the ★ icon on any poster. Clear the Saved filter to see all eligible movies." |
| Empty state — Shortlist filter active | Heading: "Shortlist is empty" · Body: "Add movies using the ✓ icon on any poster. Clear the Shortlist filter to see all eligible movies." |
| Empty state — both filters active | Heading: "No movies match" · Body: "No movies are both saved and shortlisted. Adjust your filters to see eligible movies." |
| Empty state — generic (no filter active, list is just empty) | Existing copy preserved — no change |
| Error state — save failed | "Couldn't save this movie. Check your connection and try again." |
| Error state — shortlist failed | "Couldn't update shortlist. Check your connection and try again." |
| Shortlist max reached | No toast — icon silently disabled at 6 items (pointer-events-none + opacity-40 on icon button) |
| Destructive action — Clear Shortlist | No confirmation dialog. Single-click clear via "Clear Shortlist" ghost button. Shortlist is per-step and auto-clears on movie selection; loss is low-stakes. |

## Filter Bar Layout

The filter bar in `GameSession.tsx` gains two new toggle buttons. Layout order (left to right):

1. Existing "All Movies / Unwatched Only" toggle (unchanged)
2. "Saved ★" toggle button (new)
3. "Shortlist ✓" toggle button (new)

5. Existing MovieFilterSidebar trigger (unchanged, remains rightmost)

All buttons use `size="sm"` and sit in the same `flex items-center gap-2` row as the existing filter controls. No new rows or layout containers needed.
