# Phase 06.1 Plan 03: MDBList API Key Settings + RT Score Display Summary

**One-liner:** MDBList API key field added to Settings page and RT Tomatometer scores surfaced in eligible movies table, Now Playing tile, and splash dialog using existing backend data.

## What Was Built

### Task 1 — Settings Page MDBList Card

Added `mdblist_api_key: ""` to the `emptyForm` initializer in Settings.tsx. Added a new MDBList Card section placed between the Plex card and the Sync Schedule card, following the exact same pattern as all other API key cards. The input uses `type="password"` to mask the key. Helper text reads "Optional. Enables Rotten Tomatoes scores on movies. Get your key at mdblist.com."

### Task 2 — RT Score Display

- Added `rt_score: number | null` to `EligibleMovieDTO` in `api.ts` to match the backend response field added in Plan 02.
- Added an RT column header `<th>` with `onClick={() => handleSortClick("rt")}` after the Rated (MPAA) column in the eligible movies table.
- Added `<td>` cell in each table row displaying `🍅 {movie.rt_score}%` when truthy, empty string when falsy/null.
- Added `{currentMovie.rt_score && <span>🍅 {currentMovie.rt_score}%</span>}` to the Now Playing stats block after the TMDB vote_average span.
- Added RT score `<Badge>` to splash dialog stats row after the vote_average Badge.
- Added `sort == "rt"` case to `get_eligible_movies` in `game.py` using the same null-stable two-pass pattern (`with_rt`/`without_rt` split, sort by `rt_score`, concat nulls last).

## Decisions Made

- RT scores render as "🍅 N%" when available, nothing when null — consistent with CONTEXT.md decision: no placeholder or loading state for missing RT scores.
- MDBList API key uses `type="password"` — consistent with the secret field pattern in CONTEXT.md.
- Backend sort for `rt` follows the exact null-stable two-pass approach established in Phase 4.3 for all other columns.

## Deviations from Plan

None — plan executed exactly as written. The backend sort support for "rt" was explicitly listed as a task action in Task 2 and was not yet present in game.py, so it was added as directed.
