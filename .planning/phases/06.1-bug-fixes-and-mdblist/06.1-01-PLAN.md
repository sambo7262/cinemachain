**Root cause:** The `list_sessions` endpoint (line 935-952) calls `_build_session_response(s, wa_map, current_movie_title=...)` but does NOT call `_enrich_steps_runtime` or pass `runtime_map`. The `_build_session_response` function needs `runtime_map` to compute `watched_runtime_minutes` (line 277-279). Without it, runtime is always None and `watched_runtime_minutes` stays 0.

Compare with `get_session_by_id` (line 987-1009) which correctly calls:
```python
poster_map, profile_map = await _enrich_steps_thumbnails(session.steps, db)
runtime_map = await _enrich_steps_runtime(session.steps, db)
return _build_session_response(
    session, wa_map,
    current_movie_title=...,
    poster_map=poster_map, profile_map=profile_map, runtime_map=runtime_map,
)
```

**Fix:** In `list_sessions` (line 948-952), for each session in the loop, also call `_enrich_steps_runtime` and pass the result to `_build_session_response`. Do NOT add `_enrich_steps_thumbnails` — the home page session cards don't need poster/profile thumbnails (those are only for the detail view), and adding them would add unnecessary DB queries to the list endpoint.

Change the loop body from:
```python
for s in sessions:
    wa_map = await _enrich_steps_watched_at(s.steps, db)
    out.append(_build_session_response(s, wa_map, current_movie_title=_resolve_current_movie_title(s)))
```

To:
```python
for s in sessions:
    wa_map = await _enrich_steps_watched_at(s.steps, db)
    runtime_map = await _enrich_steps_runtime(s.steps, db)
    out.append(_build_session_response(s, wa_map, current_movie_title=_resolve_current_movie_title(s), runtime_map=runtime_map))
```

Apply the same fix to `list_archived_sessions` (line 955-968) — it has the same bug.
