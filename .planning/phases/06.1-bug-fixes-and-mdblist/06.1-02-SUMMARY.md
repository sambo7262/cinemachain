# Phase 06.1 Plan 02: MDBList RT Score Backend Summary

**MDBList service fetches Rotten Tomatoes scores on-demand per movie via mdblist.com/api/, cached in Movie.rt_score with 0 sentinel for no-data, wired into get_eligible_movies with mdblist_api_key in settings API**

- **Completed:** 2026-03-22T15:22:58Z
- **Tasks:** 2
- **Files modified:** 6

## Accomplishments
- Added rt_score and rt_audience_score nullable integer columns to Movie model with Alembic migration 0008
- Created MDBList service with fetch_rt_scores that fetches from mdblist.com/api/ per tmdb_id, parses ratings array for 'tomatoes' and 'tomatoesaudience' sources
- Wired RT score enrichment into get_eligible_movies after existing TMDB enrichment, with DB refresh and 0-sentinel-to-null conversion
- Added mdblist_api_key to SettingsResponse and SettingsUpdateRequest (auto-encrypted via SECRET_KEYWORDS), plus frontend SettingsDTO

## Files Created/Modified
- `backend/app/services/mdblist.py` - New MDBList API client; fetch_rt_scores fetches and caches RT scores per movie
- `backend/alembic/versions/20260322_0008_rt_scores.py` - Migration 0008 adding rt_score and rt_audience_score to movies table
- `backend/app/models/__init__.py` - Added rt_score and rt_audience_score Mapped columns to Movie
- `backend/app/routers/game.py` - Added rt_score to EligibleMovieResponse, both movies_map dict blocks, and RT enrichment block in get_eligible_movies

- `frontend/src/lib/api.ts` - Added mdblist_api_key to SettingsDTO interface

## Decisions Made
- rt_score=0 is a sentinel meaning "fetched, RT data not available" — distinguishes from NULL (never fetched), so we avoid re-fetching on every request. Converted to null before sending to frontend.
- Synchronous per-movie fetching chosen over batching (simpler, acceptable for cold-start-only pattern since subsequent requests skip movies with rt_score IS NOT NULL).
- No changes to settings_service.py needed — mdblist_api_key auto-encrypts because the key contains "key" matching _SECRET_KEYWORDS.

To enable RT scores, users must configure the MDBList API key in Settings UI (mdblist_api_key field). The key is encrypted at rest automatically.
