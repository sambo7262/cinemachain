<decisions>
## Implementation Decisions

### Bug 1 — Settings Page Navigation Broken
- The settings icon is visible in the NavBar but clicking it does not open/navigate to /settings
- Fix: diagnose the NavBar link or App.tsx routing — the route exists (`/settings` added in 06-06) but navigation is not working
- Suspected cause: the `return null` during loading in App.tsx (while settingsStatus loads) may be swallowing the router before routes are registered, or the NavBar link has incorrect `to` prop or is not inside the Router context

### Bug 2 — Home Page Session Card Stats Showing Zeros
- Session cards on GameLobby show zeros for watched count and total runtime
- The same stats show correct values inside the GameSession page
- These fields (`watched_count`, `watched_runtime_minutes`) come from the backend `GameSessionResponse`
- Fix: investigate why the GameLobby list endpoint returns zero values while the individual session endpoint returns correct values — likely the list query is not computing aggregates, or the frontend is reading the wrong field names

### Bug 3 — Session Card Stats Should Match Stored Session Data
- This is the same issue as Bug 2 — the home page stats should reflect the actual stored session data
- Confirmed: session-level stats (inside GameSession page) are correct, so backend computes them correctly for the detail endpoint; the list endpoint may be missing the aggregate computation

### MDBList Feature — API Key in Settings
- Add `mdblist_api_key` field to the Settings page (alongside existing TMDB/Radarr/etc keys)
- The field must be marked as secret (Fernet-encrypted at rest, masked in GET response)
- Settings backend already supports arbitrary key/value pairs with encryption — just add the field to the frontend form and document the key name

### MDBList Feature — Backend RT Score Fetch + Cache
- Add `rt_score: Integer | None` and `rt_audience_score: Integer | None` columns to the `movies` table via new Alembic migration
- MDBList endpoint: `https://mdblist.com/api/?apikey={KEY}&tm={TMDB_ID}` returns JSON with `ratings` array containing `{source: "tomatoes", value: N}` for Tomatometer and `{source: "tomatoesaudience", value: N}` for Audience Score
- Fetch strategy: on-demand when movie is returned by `get_eligible_movies`, if `rt_score` is null and `mdblist_api_key` is configured; cache result in DB; do not block response (fire-and-forget or accept 1 extra call latency)
- If MDBList API key is not configured or fetch fails, `rt_score` stays null — never break movie display for missing RT data

### MDBList Feature — Frontend RT Score Display
- **Eligible movies table (filtering columns)**: add RT % column (e.g. "🍅 85%") visible when `rt_score` is not null; sortable
- **Now Playing tile stats**: add RT score alongside existing MPAA badge / runtime / TMDB rating
- **Movie splash dialog**: add RT score to the badge row (between TMDB rating and other badges)
- Display format: Tomatometer score as percentage (e.g. "85%") with 🍅 prefix when available, nothing when null — never show placeholder/loading state for RT

### Claude's Discretion
- Migration numbering (use 0008)
- Whether to fetch RT score synchronously (simple, adds latency) or as a background task (complex, no latency) — prefer synchronous for now, can optimize later
- Exact column sort direction default for RT column in table
- Whether to show audience score anywhere — not required, only Tomatometer needed per the decisions above

</decisions>
