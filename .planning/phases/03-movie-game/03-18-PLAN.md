  <how-to-verify>
    Open the app at the Tailscale IP (or localhost:3111 if testing locally). Run these five tests in sequence:

    **Test 1 — Session start guidance (GAME-01)**
    1. Go to the lobby (/)
    2. Click the "Search" tab, type a movie title (e.g., "Cast Away"), select the result
    3. Session creates and you are navigated to /game/{id}
    Expected: The session state panel shows "Watch Cast Away, then come back and pick an actor to begin the chain." — NOT "Pick an actor from the Eligible Actors tab"
    Pass if: Watch-first guidance is displayed. Eligible Actors tab is visible but guidance is to watch first.

    **Test 2 — Combined Eligible Movies on mount (GAME-03)**
    1. Immediately after Test 1 (session just created, no actor selected)
    2. Click the "Eligible Movies" tab
    Expected: Movies populate within a few seconds (TMDB fetch may take 2-5s on first load). Each row shows a "Via [Actor Name]" column. NOT an empty state telling you to select an actor.
    Pass if: At least one movie appears in the Eligible Movies tab without any actor selection.

    **Test 3 — Full move cycle + actor deduplication (GAME-04)**
    1. Using Test 1 session: mark the starting movie as watched (use the manual mark button or wait for Plex scrobble)
    2. After "marked as watched" banner appears, click "Continue the chain"
    3. Pick any actor from the Eligible Actors tab
    4. Pick any movie from the Eligible Movies tab — confirm the movie selection dialog
    5. Session advances. Mark the second movie as watched. Click "Continue the chain."
    6. Open the Eligible Actors tab for the second movie
    Expected: The actor you picked in step 3 does NOT appear in the Eligible Actors list.
    Pass if: Previously picked actor is absent.

    **Test 4 — Sort controls and watched toggle (GAME-05, GAME-06, GAME-07)**
    1. In an active session on the Eligible Movies tab with data loaded
    2. Change sort from "By Rating" to "By Runtime" — confirm row order changes
    3. Change sort to "By Genre" — confirm row order changes again
    4. Click "Unwatched Only" toggle to switch to "All Movies" — watched movies should appear with "Watched" badge
    5. Toggle back to "Unwatched Only" — watched movies disappear
    6. If a watched movie is visible in "All Movies" mode, click on it
    Expected: No confirmation dialog fires for a watched movie (it is not selectable)
    Pass if: All three sort orders work. Toggle works. Watched movies are non-clickable.

    **Test 5 — Radarr conditional notification (GAME-08)**
    This test requires picking two movies: one already in Radarr, one new.
    - Pick a movie you know is already in your Radarr library → confirm dialog → Expected: notification "Already in your library — waiting for watched event."
    - Pick a movie NOT in Radarr → confirm dialog → Expected: notification "Added to Radarr queue."
    Pass if: Notification text matches the Radarr status (no "This will add it to Radarr" unconditional message).

    **Report results for each test.** If any test fails, describe: which test, what you saw, what you expected.
