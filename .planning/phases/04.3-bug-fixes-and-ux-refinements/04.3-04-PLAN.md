**Fix 1: posterUrl null-guard.**

Replace the current `posterUrl` function:
```typescript
function posterUrl(item: PosterWallItem): string {
  if (item.poster_local_path) {
    return `/api${item.poster_local_path}`
  }
  return `https://image.tmdb.org/t/p/w185${item.poster_path}`
}
```

With a null-safe version that returns null when both paths are absent:
```typescript
function posterUrl(item: PosterWallItem): string | null {
  if (item.poster_local_path) {
    return `/api${item.poster_local_path}`
  }
  if (item.poster_path) {
    return `https://image.tmdb.org/t/p/w185${item.poster_path}`
  }
  return null  // Both paths absent ŌĆö skip this poster
}
```

**Fix 2: Guard img src against null.**

In the JSX that renders poster images (find the `<img>` element inside the map), update it to skip rendering when posterUrl returns null:

```tsx
{/* Only render if we have a valid poster URL */}
{posterUrl(poster) && (
  <img
    src={posterUrl(poster)!}
    alt=""
    className="w-full h-full object-cover"
    loading="lazy"
    onError={(e) => {
      // Hide broken images rather than showing broken image icon
      (e.target as HTMLImageElement).style.display = "none"
    }}
  />
)}
```

If the current `<img>` is inside a column-distributed layout, replace only the `<img>` tag, not the surrounding container div. The container placeholder should remain so the column layout is preserved.

**Part A ŌĆö BUG-D: Fix empty row below NavBar.**

The research confirms the App.tsx layout is:
```
NavBar
RadarrNotificationBanner   ŌåÉ returns null when no message, but may leave CSS gap
div.max-w-[1400px]
```

The `RadarrNotificationBanner` returns null when `!radarrMessage`, but the React element itself still participates in the flex/block layout as an empty slot. The empty row is caused by either:
(a) `RadarrNotificationBanner` rendering a wrapping element even when returning null ŌĆö but we've confirmed it returns literal `null`, so React renders nothing.
(b) The outer `div.min-h-screen` may have implicit spacing.
(c) The `NavBar` component itself may have a `mb-*` or the container has `pt-*`.

Approach: Wrap `RadarrNotificationBanner` in a conditional that evaluates the context value:

Read `NotificationContext.tsx` to understand how to access `radarrMessage` at the App level.

If `NotificationContext` exports `radarrMessage` via hook, use it in App.tsx to conditionally render the banner:
```tsx
// In App.tsx, this approach renders nothing at all (not even an empty element):
{/* RadarrNotificationBanner is self-contained ŌĆö renders null when no message */}
<RadarrNotificationBanner />
```

Since `RadarrNotificationBanner` already returns `null`, the empty row must come from elsewhere. Most likely cause: the outer container `div.min-h-screen` has `NavBar` as a block child, and there is a browser default margin somewhere.

**Diagnosis approach (implement during execution):**
1. Inspect `NavBar.tsx` for `mb-*` classes or default browser margins.
2. Inspect whether `div.min-h-screen.bg-background.text-foreground` has any `gap` or `pt` class.
3. Add `overflow-hidden` to the outer container if there's a margin-collapse issue.

**Most likely fix**: The outer container in App.tsx is a `div` (block element). NavBar is likely rendered as a block. The `RadarrNotificationBanner` returns `null`. Any gap is likely from NavBar having a bottom margin, OR the content div having a top padding.

Apply this fix to App.tsx outer container ŌĆö add explicit `flex flex-col` so children stack without gaps:

Also inspect `NavBar.tsx` ŌĆö if it has `mb-4` or similar, remove it (NavBar should not own its own bottom spacing).
