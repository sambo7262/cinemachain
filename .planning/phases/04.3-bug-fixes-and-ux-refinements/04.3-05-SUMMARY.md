## Failure Details

### BUG-A — FAIL: Poster wall images not visible on NAS

- **What failed:** Poster images are not visible anywhere on the live NAS. The blur overlay effect may be present but no poster images render behind it.
- **Scope:** Affects all session pages. No session on NAS shows poster images.
- **Likely root cause:** Either the `/api/static/posters/` static file serve path is not correctly mounted in the NAS Docker environment, or the `/movies/poster-wall` endpoint returns items where both `poster_local_path` and `poster_path` are null (no valid fallback URL).
- **Fix required:** Gap closure plan to investigate static file serving in NAS context and ensure fallback to TMDB CDN URL works when local poster file is absent.

### UX-A — PARTIAL FAIL: Rating sort produces wrong order

- **What worked:** Column header click registers, sort direction indicator (↑/↓) toggles correctly on click.
- **What failed:** Rating sort order is wrong. Descending shows e.g. `6.2, 6.5, 3.4, 7.5` instead of the expected `7.5, 6.5, 6.2, 3.4`.
- **Likely root cause:** The sort comparator for `rating` is comparing string values instead of numeric values, causing lexicographic sort order rather than numeric. The numbers `6.2`, `6.5`, `3.4`, `7.5` in string order would be `3.4 < 6.2 < 6.5 < 7.5`, but the observed output does not match either string or numeric order — could also be a sort stability issue or the column key mapping to the wrong field.
- **Fix required:** Gap closure plan to correct the rating sort comparator in GameSession.tsx eligible movies sort logic.
