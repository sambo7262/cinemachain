<decisions>
## Implementation Decisions

### BUG-A: Background poster wall not rendering

- **A1 — Poster images not visible behind blur:** PosterWall component is mounting (blur effect is active and darkening the page) but the poster image grid itself is not rendering. Root cause is likely one of: (a) `/movies/poster-wall` endpoint returning movies with null `poster_local_path` and null `poster_path`, (b) frontend URL construction for `/api/static/posters/{tmdb_id}.jpg` is broken, or (c) z-index stacking buries the image layer under the overlay. All three must be investigated and fixed.
- **A2 — Radarr banner affected by blur:** The `NotificationContext` banner is inside the PosterWall's blur stacking context, causing it to appear darkened and hard to read. Fix: ensure the notification banner mounts at the top of the DOM tree, outside and above the PosterWall container, so it is never subject to the blur/overlay layer.

### BUG-B: Mark as Watched causes permanent image blank-out

- Images (Now Playing card poster + chain history thumbnails) load correctly on first page visit.
- After clicking "Mark as Watched", all images go blank and do **not** recover within the same page visit — user must navigate away and return to see images again.
- Root cause: mutation invalidates the session query; refetch returns session data but either (a) `poster_path` fields come back null on steps in the refetch response, or (b) the image elements unmount/remount and fail to reload.
- Fix: use `placeholderData: keepPreviousData` on the session query so existing image data remains visible during refetch. Also investigate whether the session refetch endpoint returns enriched `poster_path` on steps (must go through `_enrich_steps_thumbnails`).

### UX-A: Column-header sorting replaces sort dropdown

- **Remove:** the sort dropdown/select control only. Genre filters and runtime range slider remain untouched.
- **Add:** clickable sort headers on columns: **Rating**, **Year**, **Runtime**, **MPAA**. Click once = ascending, click again = descending.
- **Default sort on load:** TMDB Rating descending (highest rated first).
- **Sort indicator:** active sort column shows ↑ (ascending) or ↓ (descending) arrow next to the header label.

### UX-B: Load More appends, scrolls to new results, hides when exhausted

- Clicking Load More appends new results to the bottom of the existing list — does not replace it.
- After appending, the view scrolls subtly to the first newly added result so the user knows where new movies begin.
- The Load More button disappears entirely when no further pages exist (all results loaded).
- When search is active or a filter is applied, pagination is replaced entirely by a full result set (no Load More in search/filter mode — see BUG-C).

### UX-C: Session stats simplified to 3 fields

- **Keep:** Watched, Runtime (total combined runtime of all movies watched in the session), Started.
- **Remove:** Steps (chain length) and Actors (unique actors used).
- No changes to the layout, styling, or behavior of the three retained stats — removal only.

### BUG-C: Search queries full filmography, bypasses pagination

- When a search term is entered, the request goes to the backend (not client-side filtering).
- Before querying, backend calls `_ensure_actor_credits_in_db` for the current actor — this is a no-op if the actor is already cached (short-circuit from 04-08), or triggers a TMDB fetch to fill gaps if not. This ensures search covers the full filmography, not just what's already loaded in the paginated view.
- Backend returns all matching results filtered by title. No pagination on search results.
- Search results replace the paginated list entirely — Load More button is hidden while search is active.
- Same behavior as filters: when any filter is active, all matching results are returned without pagination.
- Clearing the search term returns to the normal paginated view.

### BUG-D: Narrow empty row below NavBar + home button pastel colors

- **Empty row:** A narrow full-width empty container renders directly below the NavBar on all pages except the app home page. It appears to be an empty placeholder div or notification slot in the main content layout that renders with height even when empty. Fix: locate the element in the layout and either remove it or conditionally render it only when it has content.
- **Home session card buttons:**
  - **Continue** button: soft pastel green background (not electric or bold — muted tint, e.g. `bg-green-100 text-green-800 border-green-200` or equivalent).
  - **Archive** button: soft pastel red background (e.g. `bg-red-100 text-red-800 border-red-200` or equivalent).
  - Button text labels unchanged. Only background/border color changes.

### Claude's Discretion

- Exact pastel color values (specific Tailwind classes) — use `*-100` background with `*-800` text and `*-200` border as a starting point; adjust if visually too light or too dark against the card background.
- Scroll-to-new-result implementation detail — use a `ref` on the first newly appended item and call `scrollIntoView({ behavior: 'smooth', block: 'nearest' })`.
- `keepPreviousData` implementation — use React Query v5 `placeholderData: keepPreviousData` import (not the v4 `keepPreviousData: true` option).

</decisions>
