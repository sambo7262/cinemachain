## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / table cell / label / column header / caption | 14px (text-sm) | 400 (regular) | 1.5 |
| Heading (page title, dialog title) | 16px (text-base) | 600 (font-semibold) | 1.2 |
| Display (brand / logo) | 18px (text-lg) | 600 (font-semibold) | 1.2 |

Notes:
- Body line height 1.5 declared via `leading-relaxed` on overview text in splash dialog
- 3 sizes used in this phase: 14px, 16px, 18px
- Column headers use 14px weight 400 with `text-muted-foreground` for visual differentiation from body cells — no additional weight tier needed
- Caption and badge text use 14px (`text-sm`) with `text-muted-foreground` — no 12px tier
- Display role (18px) uses 600 (semibold); brand/logo rendering at NavBar level is outside this phase's implementation scope

## Interaction Spec

### Search Input
- Placeholder text: `Search movies... or "a: Tom Hanks" for filmography`
- Syntax reference below input (always visible, not a tooltip): `m: title · a: actor · d: director`
  - Render as `<p className="text-sm text-muted-foreground">` immediately below the input
- Debounce: 400ms before triggering API call (match existing GameSession debounce pattern)
- On focus: no special state — standard shadcn Input focus ring
- Clear button: show an X icon inside the input when value is non-empty (lucide `X`, 14px); `aria-label="Clear search"`

Source: CONTEXT.md D-01

### Genre Chips (landing state)
- Show when: search input is empty AND no genre has been selected
- Layout: `flex flex-wrap gap-2`
- Each chip: `Button variant="outline" size="sm"` with genre label
- Active/loading chip: switch to `variant="default"` (primary fill) while results are loading and once selected
- Chip list (curated, fixed): Action, Adventure, Animation, Comedy, Crime, Documentary, Drama, Fantasy, Horror, Romance, Science Fiction, Thriller, History, Music (14 genres)
- Clicking a chip replaces the landing state with the results table; chip remains highlighted to show active genre context

Source: CONTEXT.md D-03

### Results Table
- Exact port of GameSession eligible movies table pattern
- Columns: poster thumbnail (48×72px), title + watched badge, year, TMDB rating (amber), RT score, MPAA rating, runtime, genre badges
- Genre column: space-separated genre names in `text-muted-foreground text-sm` (not present in GameSession table — new column for Search page)
- Sort: click column header to toggle asc/desc; active sort direction shown with `▲` / `▼` inline text appended to header label
- Default sort: rating descending
- Row click: opens movie splash dialog
- Row hover: `hover:bg-accent/50 cursor-pointer`
- Watched/All toggle: `Button variant="outline" size="sm"` pair above table right-aligned — "All" and "Unwatched Only"; active button uses `variant="default"`
- Pagination: same component/pattern as GameSession (prev/next, page indicator) — page size 20

Source: CONTEXT.md D-02; GameSession.tsx lines 900–998

### Movie Splash Dialog (Search context)
- Reuse exact structure from GameSession: `max-w-2xl Dialog`
- Left column: poster `w-32 rounded-md` (w185 TMDB image)
- Right column: badges row (TMDB rating, RT, MPAA, runtime, year) + overview text + TMDB external link
- Action buttons replace game-specific buttons:

| Button | Variant | Label (default) | Label (loading) | Label (success) |
|--------|---------|-----------------|-----------------|-----------------|
| Primary | `default` | "Download via Radarr" | "Requesting..." | "Added to Radarr" |
| Secondary | `outline` | "Watch Online" | "Saving..." | "Marked as Watched" |
| Cancel | `outline` | "Close" | — | — |

- After either action button fires successfully: button label switches to success state for 2 seconds, then returns to default. Dialog remains open — user closes manually via "Close" or dialog backdrop.
- If Radarr returns `already_in_radarr`: show label "Already in Radarr" (informational, no error state)
- If Radarr returns `error`: show `<p className="text-sm text-destructive">` error message inline below the buttons — "Request failed. Try again or check Radarr."

Source: CONTEXT.md D-04; GameSession.tsx lines 1043–1141

### Loading States
- Search results loading: table skeleton — same `animate-pulse` pattern used in GameSession; show 5 skeleton rows of equal height to content rows
- Genre chips loading (after click): active chip switches to `variant="default"` + adds `animate-pulse` class until results arrive
- Splash dialog action loading: disable button, swap label to loading text (see table above)

### Empty States
- Search returns no results: `<p className="text-sm text-muted-foreground py-8 text-center">` — copy per table below
- Genre browse returns empty (unexpected): same paragraph element — copy per table below

## Accessibility Contract

- Search input: `aria-label="Search movies, actors, or directors"`
- Search clear button (lucide X icon): `aria-label="Clear search"`
- Genre chips: `aria-label="{genre} movies"` on each Button
- Watched toggle buttons: `aria-pressed={true/false}` on active state
- Results table: `<table>` with `<thead>` / `<tbody>`; `<th scope="col">` on all header cells
- Sort headers: `aria-sort="ascending"` / `aria-sort="descending"` / `aria-sort="none"` on active sort column
- Splash dialog: `DialogTitle` contains movie title; `DialogDescription` contains year + MPAA (for screen reader context)
- Action buttons in splash: descriptive labels include movie title when in loading/success state is not needed — default labels are sufficient
- External TMDB link: `aria-label="View {title} on TMDB (opens in new tab)"` (matches existing pattern)
- Touch targets: all interactive elements minimum 44px touch target (genre chips use `size="sm"` Button which renders at 32px height — wrap with `min-h-[44px] flex items-center` on mobile if needed, or use `size="default"` on mobile breakpoint)
