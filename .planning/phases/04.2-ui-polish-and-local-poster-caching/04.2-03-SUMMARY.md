**GET /movies/poster-wall endpoint serving tmdb_id/poster_path/poster_local_path tuples, plus _download_posters_pass() that downloads TMDB posters to local disk nightly and updates Movie.poster_local_path in DB**

## Accomplishments
- GET /movies/poster-wall endpoint: Priority 1 from WatchEvent history, Priority 2 supplement from popular DB movies; returns list[PosterWallItem] with tmdb_id, poster_path, poster_local_path
- _download_posters_pass(): queries movies with poster_path set but poster_local_path NULL, downloads each at w185 tier via existing httpx AsyncClient, writes to static/posters/{tmdb_id}.jpg via run_in_executor, updates DB, swallows per-poster errors as warnings
- Route ordering enforced: /poster-wall declared before /{tmdb_id} catch-all to prevent FastAPI treating "poster-wall" as integer tmdb_id
- Full backend test suite green: 20 passed, 55 skipped locally (asyncpg skips expected on local dev)

## Decisions Made
- PosterWallItem `poster_local_path: str | None = None` — None signals poster not yet downloaded; frontend falls back to TMDB CDN path
- Supplement trigger is `< 20` watched posters (not `< limit`) — ensures meaningful wall even for new users with few watch events
- `run_in_executor` for file writes — aiofiles is not in requirements.txt; run_in_executor is the correct async-safe pattern
- Race condition guard added: if file exists on disk but DB not updated, do DB-only update and continue — guards against partially-completed runs
