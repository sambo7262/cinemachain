## Summary

Phase 4.2 is a well-scoped enhancement pass with no new game mechanics. Every decision is locked in CONTEXT.md and the UI-SPEC is fully approved. The backend surface is small: one new column (`poster_local_path`), one new migration (0006 chains from 0005), one new nightly pass in `cache.py`, one `StaticFiles` mount in `main.py`, three new fields on `GameSessionResponse`/`GameSessionDTO`, and one new endpoint (`GET /movies/poster-wall`). The frontend surface is also bounded: a new `PosterWall.tsx` component, in-place extensions to `SessionCounters.tsx`, `GameSession.tsx`, and `GameLobby.tsx`.

The poster-wall animation is pure CSS `@keyframes` with GPU-composited `transform: translateY` — no JavaScript scroll loop. The column duplication trick (poster list × 2 end-to-end per column) creates a seamless infinite scroll without JavaScript position resets. This is the canonical approach for CSS infinite ticker animations and is supported in every evergreen browser.

The most implementation-sensitive piece is the Docker volume for static posters. The Synology NAS pattern already established in `compose.yaml` uses host bind-mounts at `/volume1/docker/appdata/cinemachain/`. The static posters volume must follow the same pattern so files survive container restarts and are visible in DSM File Station. FastAPI's `StaticFiles` mount must be added before `yield` in `main.py`'s `lifespan` function — the mount registration itself is synchronous and does not depend on any awaitable.

**Primary recommendation:** Work in three independent streams (backend data, backend API, frontend) with the Alembic migration as the first committed artifact since every other backend task depends on the column existing in the live DB.

### Pattern 1: CSS Infinite Scroll Animation (Poster Wall)

**What:** Duplicate the poster list end-to-end in each column div. Animate translateY from 0% to -50% (scrolling up) or -50% to 0% (scrolling down). At -50%, the visual position is identical to 0% because the second copy starts where the first ended — creating a seamless loop.

**When to use:** Any time you need an infinite ticker/scroll without JavaScript.

**Column structure:**
```tsx
// Source: CSS animation pattern (verified against MDN keyframes spec)
// Each column: poster array duplicated [arr, arr] → seamless loop

const PosterWall = ({ posters }: { posters: PosterItem[] }) => {
  // Responsive column count: 2 (sm), 3 (md), 4 (lg+)
  // Build columns by distributing posters round-robin
  return (
    <div className="fixed inset-0 -z-10 overflow-hidden">
      {/* Dark overlay */}
      <div className="absolute inset-0 bg-black/80 z-10" />
      {/* Columns container */}
      <div className="flex h-full gap-0">
        {columns.map((col, i) => (
          <div
            key={i}
            className={cn(
              "flex flex-col gap-0 animate-poster-scroll",
              i % 2 === 0 ? "direction-normal" : "direction-reverse"
            )}
            style={{ willChange: "transform" }}
          >
            {/* Duplicate list for seamless loop */}
            {[...col, ...col].map((poster, j) => (
              <img key={j} src={posterUrl(poster)} alt="" className="w-full object-cover blur-[20px]" />
            ))}
          </div>
        ))}
      </div>
    </div>
  )
}
```

**CSS keyframes (in index.css or Tailwind config):**
```css
/* Source: CSS @keyframes + Tailwind animation extension pattern */
@keyframes poster-scroll-up {
  from { transform: translateY(0); }
  to   { transform: translateY(-50%); }
}
@keyframes poster-scroll-down {
  from { transform: translateY(-50%); }
  to   { transform: translateY(0); }
}
```

**Tailwind config extension:**
```js
// tailwind.config.js — add to theme.extend
animation: {
  'poster-up':   'poster-scroll-up 75s linear infinite',
  'poster-down': 'poster-scroll-down 75s linear infinite',
}
```

**75 seconds** is the value specified in the UI-SPEC (within the CONTEXT.md 60-90s target).

### Pattern 2: FastAPI StaticFiles Mount

**What:** Mount a local filesystem directory to a URL path so the frontend can fetch files via nginx proxy.

**When to use:** Serving locally cached files that bypass external CDN.

```python
# Source: FastAPI official docs — StaticFiles
from fastapi.staticfiles import StaticFiles
import os

# In lifespan, before yield — or at module level
os.makedirs("static/posters", exist_ok=True)
app.mount("/static", StaticFiles(directory="static"), name="static")
```

**Critical detail:** `StaticFiles` raises `RuntimeError` at mount time if the directory does not exist. The `os.makedirs(..., exist_ok=True)` call must precede the mount, or the nightly job must guarantee directory creation before `app.mount` is called. Safest: create the directory in the lifespan startup block.

**nginx proxy configuration:** The frontend container proxies `/api/` to the backend. Static files at `/static/` on the backend are accessed as `/api/static/` from the frontend perspective via the existing nginx proxy. Verify the nginx config handles `/api/static/` correctly — or mount at `/api/static/` directly to keep URL paths consistent. Check `frontend/nginx.conf` for the proxy rule before implementing.

### Pattern 4: Poster Download Pass in Nightly Job

**What:** After the existing movie enrichment passes in `cache.py`, add a new pass that downloads posters for movies with `poster_path IS NOT NULL` and `poster_local_path IS NULL`.

```python
# Source: httpx streaming pattern (verified — TMDBClient already uses httpx.AsyncClient)
import os, httpx

async def _download_poster(tmdb_id: int, poster_path: str, dest_dir: str, client: httpx.AsyncClient) -> str | None:
    url = f"https://image.tmdb.org/t/p/w185{poster_path}"
    dest = os.path.join(dest_dir, f"{tmdb_id}.jpg")
    if os.path.exists(dest):
        return dest  # already cached — skip
    try:
        r = await client.get(url, timeout=30)
        r.raise_for_status()
        # Write synchronously (small file — 185px poster ~20KB) or use aiofiles
        with open(dest, "wb") as f:
            f.write(r.content)
        return dest
    except Exception as exc:
        logger.warning("poster download failed for tmdb_id=%d: %s", tmdb_id, exc)
        return None
```

**Rate limiting:** TMDB allows 40 req/s. The existing `asyncio.sleep(0.05)` pattern in `nightly_cache_job` applies here too — add the same sleep between poster downloads.

**DB update:** After a successful download, `UPDATE movies SET poster_local_path = $1 WHERE tmdb_id = $2`. Use `sa.update(Movie)` within the existing `_bg_session_factory` session pattern.

### Pattern 6: Poster-Wall Backend Endpoint

**What:** `GET /movies/poster-wall?limit=40` returns `{tmdb_id, poster_path, poster_local_path}` list.

**Query logic:**
1. SELECT distinct tmdb_id from WatchEvents, join Movie for poster_path and poster_local_path, WHERE poster_path IS NOT NULL. LIMIT 40.
2. If len(results) < 20: supplement by querying the movies table for popular movies (vote_count desc), excluding already-collected tmdb_ids. OR make a live TMDB /movie/popular call and cross-reference against Movie rows in DB. **Prefer DB-only supplementation** (no TMDB live call at request time) — query movies with vote_count IS NOT NULL ORDER BY vote_count DESC, exclude already-in-list tmdb_ids, LIMIT needed.

**New endpoint location:** `backend/app/routers/movies.py` — already has a `/movies/` router; add there to avoid cluttering `game.py` further.

**Response schema:**
```python
class PosterWallItem(BaseModel):
    tmdb_id: int
    poster_path: str
    poster_local_path: str | None = None
```

### Anti-Patterns to Avoid

- **Animating with JavaScript:** Do NOT use scroll events, `requestAnimationFrame`, or `setInterval` for the poster drift. Pure CSS is both simpler and more performant — especially on NAS hardware.
- **Blocking event loop during file I/O:** Do NOT use synchronous `open()` inside an `async def` in the nightly job without an executor wrapper. Either use aiofiles or `run_in_executor`.
- **Directory creation at app startup without exist_ok:** The `/static/posters/` directory may not exist on a fresh deploy. `os.makedirs(..., exist_ok=True)` prevents a crash on first boot.
- **Storing absolute host paths in DB:** `poster_local_path` should store the relative path `/static/posters/{tmdb_id}.jpg` (the URL-relative form), not the filesystem path `/app/static/posters/...`. This keeps the value portable and directly usable as a frontend URL.
- **Conflating step_count with steps.length:** "Chain steps" in the UI means actor picks (steps where actor_tmdb_id IS NOT NULL), not total step rows. The first step (starting movie, no actor) should not count.

## Common Pitfalls

### Pitfall 1: StaticFiles directory not created before mount
**What goes wrong:** FastAPI raises `RuntimeError: directory "static" does not exist` at startup when the volume is missing or the path doesn't exist on the first deploy.
**Why it happens:** Docker bind-mount creates the path on the host but the container path may not exist until the first container start writes to it.
**How to avoid:** Add `os.makedirs("static/posters", exist_ok=True)` in the lifespan startup block before `app.mount(...)`.
**Warning signs:** Container crashes on startup with RuntimeError in logs.

### Pitfall 2: nginx not proxying /static/ path
**What goes wrong:** Frontend fetches `/static/posters/123.jpg` but nginx doesn't forward it to the backend — returns 404.
**Why it happens:** The nginx proxy in the frontend container currently routes `/api/` to the backend. The `/static/` prefix is not `/api/`, so it won't be proxied unless the nginx config is updated.
**How to avoid:** Two options: (a) mount FastAPI static at `/api/static/` so the existing `/api/` proxy rule covers it, or (b) add a separate `/static/` location block to nginx.conf. Read `frontend/nginx.conf` before deciding — option (a) is simpler and requires no nginx change.
**Warning signs:** Browser dev tools show 404 on `/static/posters/...` requests.

### Pitfall 3: Poster wall renders on non-session pages
**What goes wrong:** PosterWall component is mounted globally (e.g. in App.tsx) and appears on GameLobby or other pages.
**Why it happens:** Developer adds the component to a layout wrapper instead of the GameSession page.
**How to avoid:** Mount PosterWall inside `GameSession.tsx` only — the root div of GameSession, not in App.tsx.

### Pitfall 4: Animation jank on NAS hardware
**What goes wrong:** Poster wall animation stutters or drops frames on the Synology NAS (limited GPU/CPU).
**Why it happens:** Animating properties other than `transform` and `opacity` forces layout recalculation. Blur applied to individual images instead of the column container doubles GPU compositing cost.
**How to avoid:** Apply `blur-[20px]` to the column container div (or the whole poster wall wrapper), not to individual `<img>` elements. Use `will-change: transform` on each scrolling column div. CSS `transform: translateY` is GPU-composited and does not trigger layout.

### Pitfall 5: Poster wall flicker on initial load
**What goes wrong:** The poster wall briefly shows a white flash or unstyled content before the animation starts.
**Why it happens:** Images load asynchronously; the animation starts before images are ready.
**How to avoid:** Use `object-fit: cover` on all poster images with a fixed height. Set a background color (`bg-muted`) on each image element so the column never appears white. The heavy blur means partial loading is invisible.

### Pitfall 6: Dead-end detected but ineligible actors still shown
**What goes wrong:** The dead-end state fires but the "Already picked" ineligible actors section is still visible beneath it, creating conflicting UI.
**Why it happens:** The dead-end empty state replaces only the eligible actors table, not the entire actors tab content.
**How to avoid:** Dead-end condition should replace the entire actors tab content area — both the eligible section and the ineligible section. Check: `eligibleActors.length === 0 && eligibleActorsData.filter(a => a.is_eligible === false).length > 0` (there ARE ineligible actors but NONE are eligible) is the dead-end condition. If `eligibleActors.length === 0` AND there are zero ineligible actors, that's the existing "no actors found" state (not a dead-end).

### Pitfall 7: Random pick ignores active filters
**What goes wrong:** The random pick button selects from `allEligibleMovies` instead of `filteredMovies`, so it may pick a movie that the user has filtered out.
**Why it happens:** Developer accesses the raw API data array instead of the client-side filtered array.
**How to avoid:** `Math.floor(Math.random() * filteredMovies.length)` — always sample from `filteredMovies`, not `allEligibleMovies`.

### Pitfall 8: Docker volume path not matching NAS convention
**What goes wrong:** Posters are lost on container restart because the volume is not a host bind-mount.
**Why it happens:** Using a named Docker volume instead of a host bind-mount path.
**How to avoid:** Follow the established pattern in `compose.yaml` — use a host bind-mount: `/volume1/docker/appdata/cinemachain/static:/app/static`. Named volumes on Synology are less visible and harder to back up.

### Session Age Formatter
```typescript
// Client-side, no backend changes needed
function formatSessionAge(createdAt: string): string {
  const diffMs = Date.now() - new Date(createdAt).getTime()
  const diffMinutes = Math.floor(diffMs / 60000)
  if (diffMinutes < 60) return `${diffMinutes}m ago`
  const diffHours = Math.floor(diffMinutes / 60)
  const remainingMinutes = diffMinutes % 60
  if (diffHours < 24) {
    return remainingMinutes === 0 ? `${diffHours}h ago` : `${diffHours}h ${remainingMinutes}m ago`
  }
  const diffDays = Math.floor(diffHours / 24)
  return `${diffDays}d ago`
}
```

## Open Questions

1. **nginx proxy path for /static/**
   - What we know: the existing nginx config forwards `/api/` to `backend:8000`. The FastAPI StaticFiles mount will be at `/static/` on the backend.
   - What's unclear: does nginx in the frontend container have a location block for `/static/` in addition to `/api/`? If not, the frontend can't fetch `/static/posters/...` directly.
   - Recommendation: read `frontend/nginx.conf` before implementing. If no `/static/` location exists, mount FastAPI static at `/api/static/` instead (simpler — no nginx change needed).

2. **aiofiles availability**
   - What we know: the nightly job runs in async context. Sync `open()` blocks the event loop.
   - What's unclear: whether `aiofiles` is in `backend/requirements.txt` already.
   - Recommendation: check `backend/requirements.txt` first. If absent, use `asyncio.get_event_loop().run_in_executor(None, write_fn)` to avoid adding a new dependency. Poster files are small (~20KB), so run_in_executor overhead is acceptable.

3. **created_at UTC vs local time**
   - What we know: `GameSession.created_at` uses `datetime.utcnow` (UTC). The session age formatter runs client-side using `Date.now()` (local time).
   - What's unclear: whether the DB-stored value has timezone info attached or is naive UTC.
   - Recommendation: when serializing `created_at` in the Pydantic model, add `model_config` with `json_encoders` or use `.isoformat() + "Z"` so the frontend can parse it unambiguously. The formatter should handle both `"Z"` suffix and bare ISO strings.
