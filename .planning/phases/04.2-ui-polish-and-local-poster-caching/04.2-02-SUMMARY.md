## Decisions Made

- `poster_local_path` is nullable — most movies on first deploy will not have a locally cached poster yet; the poster download worker (04.2-03) fills this in asynchronously.
- StaticFiles mount is inside lifespan before `yield` so the `static/posters/` directory is guaranteed to exist before FastAPI validates it on mount.
- `step_count` counts only actor-pick steps (`actor_tmdb_id IS NOT NULL`) because the first step (starting movie) has a null actor; this matches the frontend's notion of "how many actors have been chained".
- `created_at` is passed from the existing `GameSession.created_at` DateTime column — no new DB column needed.
