## Color

The project uses shadcn zinc CSS variable tokens. Dark mode is enabled. All color references must use CSS variable tokens, not raw hex.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` / `hsl(var(--background))` | Page backgrounds, NavBar, session page root |

## Interaction Contract

### Visual Hierarchy ‚Äî GameSession Main Screen

Primary focal point: the Now Playing card ‚Äî largest card on screen, centered in the main content area, no competing elements at the same visual weight. All other UI elements (SessionCounters bar, tab nav, action buttons) are subordinate in size and color weight to the Now Playing card.

### Poster Wall Animation

- CSS `@keyframes` only ‚Äî no JavaScript scroll logic.
- Columns 1 and 3 scroll upward (`translateY` from 0 to -50%); columns 2 and 4 scroll downward (`translateY` from -50% to 0).
- Animation duration: 75 seconds per full cycle for all columns (within CONTEXT.md's 60‚Äì90s target).
- `animation-timing-function: linear` ‚Äî constant drift, not eased.
- `animation-iteration-count: infinite`.
- Each column contains the poster list duplicated once end-to-end (seamless loop without JavaScript).
- `will-change: transform` on each column for GPU compositing on NAS hardware.
- Fallback: if fewer than 5 posters are available, the poster wall is not rendered; `bg-background` class remains on the root div.

### Random Pick Button

- Placement: Eligible Movies tab header row, right-aligned alongside the tab section title.
- Label: "Random" with a Shuffle icon (lucide `Shuffle`).
- Variant: `ghost` with `size="sm"`.
- Disabled state: `opacity-50 pointer-events-none` when eligible movie list is empty or no actor has been selected.
- On click: opens shadcn `Dialog` (existing `dialog.tsx` primitive).
  - Dialog title: "Random Pick"
  - Dialog body: movie title at heading size, year and runtime at body size. Poster thumbnail at `w-16 h-24` (matching MovieCard poster dimensions) if `poster_path` is available; `bg-muted` placeholder at identical dimensions if null.
  - Dialog footer: ["Keep Browsing"] (variant `outline`) and ["Request This Movie"] (variant `default`).
- On confirm: calls `request_movie`, advances chain, closes dialog, triggers Radarr notification via `useNotification()`.

### Dead-End Detection

- Trigger: Eligible Actors tab renders with zero eligible actors.
- Replaces the empty actor grid ‚Äî do not show the normal "No eligible actors" text.
- Empty state layout: centered vertically in the tab content area.
  - Heading: "Chain dead end" at heading type size, foreground color.
  - Body: "All actors from this movie have been used in this session." at body type size, muted-foreground color.
  - Actions row: two buttons side-by-side, 8px gap.
    - [Delete Last Step] ‚Äî variant `outline`, triggers existing `deleteStepOpen` Dialog flow.
    - [End Session] ‚Äî variant `destructive`, triggers the existing archive/end flow.
- No badge, indicator, or proactive warning on session home view.

### Session Stats Expansion ‚Äî SessionCounters Component

The existing `SessionCounters.tsx` component is expanded in-place. Props interface gains three new fields.

Five stats displayed in a single row with `|` separator dividers between each:

| Stat | Label | Format |
|------|-------|--------|
| Movies watched | "Watched" | integer count |
| Total runtime | "Runtime" | Xh Ym (existing `formatRuntime` helper) |
| Chain steps | "Steps" | integer count |
| Unique actors | "Actors" | integer count |
| Session age | "Started" | "Xh Ym ago" for <24h; "Xd ago" for ‚â•24h |

Layout: each stat is a `flex flex-col items-center` block with `dt` (label) above and `dd` (value) below ‚Äî matching the existing pattern. Dividers use `h-8 w-px bg-border` between each pair (existing pattern).

"Session age" is computed client-side from `session.created_at` ‚Äî no new backend field needed; `GameSessionDTO` already returns `created_at`.

### Session Cards in GameLobby

Each active session card gains a compact stats line below the session name.

Stats to display: movies watched ¬∑ chain steps ¬∑ started X ago.

Format: `{watchedCount} watched ¬∑ {stepCount} steps ¬∑ started {age}` ‚Äî single line, `text-xs text-muted-foreground`.

Backend must return `step_count` and `unique_actor_count` in `GameSessionDTO` (see integration point in CONTEXT.md). `created_at` is already returned.
