## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, badge padding, inline spacing |
| sm | 8px | Compact element spacing, input padding, stat gaps |
| md | 16px | Default element spacing, card padding, modal content padding |
| lg | 24px | Section padding, dialog content vertical rhythm |
| xl | 32px | Layout gaps, splash modal internal padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (onboarding screen) |

Exceptions:
- Touch targets (icon buttons): minimum 44px hit area — apply `min-w-[44px] min-h-[44px]` to icon-only buttons
- Poster image in Item 2 movie splash: `w-32` (128px) fixed width, height proportional

Source: derived from existing codebase patterns (px-4, py-3, gap-2, gap-4) in `GameSession.tsx` and `GameLobby.tsx`

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 regular | 1.5 | `text-sm` |
| Label | 12px | 400 regular | 1.4 | `text-xs` |
| Subheading | 16px | 600 semibold | 1.4 | `text-base font-semibold` |
| Heading | 20px | 600 semibold | 1.2 | `text-xl font-semibold` |

Notes:
- Exactly 2 font weights in use: 400 regular and 600 semibold. No 500 medium or 700 bold for new Phase 6 additions.
- Page-level title (`CinemaChain` on home page) uses `text-3xl font-bold tracking-tight` — retain as-is, not a new addition
- All new copy in Phase 6 uses body (14px/400) or subheading (16px/600) as the default
- Movie splash overview text uses body (14px/400) at 1.5 line-height — no truncation (D-05)
- Settings field labels use label (12px/400) — the smaller size alone provides sufficient visual distinction from body
- Do NOT introduce font sizes or weights outside this set in Phase 6

Source: `GameLobby.tsx` lines 216, 236, 241; `GameSession.tsx` lines 405–409, 509

## Color

The project uses shadcn zinc dark mode CSS variables. All values reference CSS custom properties — not raw hex.

| Role | CSS Variable | Dark Mode Computed | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `hsl(var(--background))` | ~hsl(240, 10%, 3.9%) near-black | Page background, dialog overlays |
| Secondary (30%) | `hsl(var(--card))` / `hsl(var(--secondary))` | ~hsl(240, 10%, 3.9%) / ~hsl(240, 3.7%, 15.9%) | Cards, dropdowns, nav, sidebar panels |
| Accent (10%) | `hsl(var(--primary))` | ~hsl(0, 0%, 98%) near-white | Primary CTAs only (see reserved list below) |
| Muted text | `hsl(var(--muted-foreground))` | ~hsl(240, 5%, 64.9%) | Descriptive/secondary text, timestamps, stat labels |
| Destructive | `hsl(var(--destructive))` | ~hsl(0, 62.8%, 30.6%) dark red | "Archive Session" confirm button only |

Accent (`--primary`) reserved for:
1. Primary action buttons (`Button` default variant) — e.g. "Save Settings", "Add to Session" in movie splash
2. Active/selected state indicators (selected movie or actor highlight)
3. `ring` focus outline on focused inputs

Accent is NOT used for:
- TMDB external link icons (use `text-muted-foreground` with `hover:text-foreground`)
- Inline stats badges (use `bg-primary/10 text-primary` for soft treatment)
- Dropdown menu items (use default menu item styling)

Semantic color exceptions for this phase:
- Required field validation error: `text-red-500` (existing pattern from `GameLobby.tsx` line 334)
- Radarr/Sonarr status banners: `text-amber-400` (existing pattern — retain)
- Error banners: `border-red-700 bg-red-950/40 text-red-400` (existing pattern — retain)

Source: `frontend/src/index.css` CSS variable definitions, sampled component usage in `GameLobby.tsx` and `GameSession.tsx`

## Component Contracts

### Item 2 — Movie Selection Splash (Dialog)

Layout within `DialogContent`:
- Left column (fixed `w-32`): TMDB poster image, `object-fit: cover`, rounded-md corners
- Right column (flex-1): Title (subheading 16px/600), TMDB rating + MPAA rating + runtime as Badge row, full overview text (body 14px/400, no truncation), TMDB external link icon
- Below columns (full width): Radarr Checkbox + label
- `DialogFooter`: "Keep Browsing" (outline variant) + "Add to Session" (default variant, primary)

Dialog width: `max-w-2xl` — wider than default to accommodate poster + overview text.

### Item 3 — Edit Session Name Modal (Dialog)

Layout within `DialogContent`:
- Single `Input` field pre-populated with current session name
- `DialogFooter`: "Discard Changes" (outline variant) + "Save Name" (default variant)

Validation: show `text-red-500 text-xs` below input if field is empty on submit.

### Item 4 — Chain History Search (Input)

Position: above the chain history table, full-width, `mb-4` margin below.
Icon: `Search` lucide icon in left slot of Input.
Behavior: filter on every keystroke (no debounce needed — client-side only).
Empty state: centered paragraph `text-sm text-muted-foreground` below the table area.

### Item 5 — TMDB External Link Icon

Use `ExternalLink` from lucide-react, size `w-3 h-3`.
Render inline after movie title or actor name text: `gap-1 items-center`.
Color: `text-muted-foreground hover:text-foreground transition-colors`.
Wrap in `<a href="..." target="_blank" rel="noopener noreferrer">`.

### Item 6 — Settings Page (`/settings` route)

Layout: full-page route at `/settings`, accessible via NavBar link with `Settings` icon.
Structure:
- Page heading "Settings" (heading 20px/600)
- Grouped field sections using `Card` + `CardHeader` + `CardContent`
- Section groups: TMDB (required), Radarr, Sonarr, Plex, Sync Schedule
- Each section uses `CardHeader` with section name as `CardTitle` (subheading 16px/600)
- Fields: `Input` with `label` above and description `text-xs text-muted-foreground` below
- Save button: full-width on mobile, right-aligned on desktop, `Button` default variant "Save Settings"

Onboarding (blocking) state:
- Full-viewport centered layout (not a route — rendered over the app if TMDB credentials absent)
- `Card` with `max-w-md` centered, heading + body copy, TMDB API key field, TMDB base URL field, "Save and Continue" CTA

NavBar addition:
- Add `<Settings className="w-4 h-4" />` icon link to NavBar, right-aligned
- Tooltip or `aria-label`: "Settings"

### Item 8 — Session Card Stats (GameLobby)

Inline stat row below session name, using `text-xs text-muted-foreground` for all three stats.
Format: `{N} watched · {H}h {M}m · Started {Month D, YYYY}` — middle-dot separator.
No separate row or expand — all three inline on a single line.

### Item 9 — Now Playing Tile Stats

Add to the existing Now Playing section in `GameSession.tsx`:
- Stat row: `text-xs text-muted-foreground` with flex gap-3
- MPAA: `Badge` variant outline, `text-xs`
- Runtime: plain text `text-xs text-muted-foreground`
- TMDB rating: `Star` lucide icon `w-3 h-3` + numeric text, `text-xs`

## Interaction States

All interactive elements in Phase 6 must handle the following states:

| State | Treatment |
|-------|-----------|
| Default | Standard shadcn component default |
| Hover | shadcn built-in hover styles — do not override |
| Focus | `ring-2 ring-ring ring-offset-2` — shadcn default |
| Disabled | `opacity-50 cursor-not-allowed` — shadcn default |
| Loading (settings save) | Button shows `animate-spin` spinner icon, disabled during request |
| Error (settings fields) | `text-red-500 text-xs` below affected input |
| Success (settings save) | Brief inline confirmation: "Settings saved." below Save button, `text-sm text-muted-foreground` |

## Accessibility Baseline

- All Dialog components must have `DialogTitle` (visible or via `sr-only`) — required by Radix
- All icon-only buttons must have `aria-label`
- TMDB external links must include `aria-label` with movie/actor name and "opens in new tab"
- Radarr checkbox must be associated with its label via `htmlFor` / `id`
- Settings inputs must have `<label>` elements with `htmlFor` matching input `id`
- Focus trap is automatic via Radix Dialog — no custom implementation needed
- Color contrast: zinc dark theme meets WCAG AA for `--foreground` on `--background` (confirmed by shadcn zinc preset)
