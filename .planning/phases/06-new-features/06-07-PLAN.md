<task type="checkpoint:decision" gate="blocking">
  <name>Task 1: Rotten Tomatoes research decision (Item 7)</name>
  <decision>Whether to implement Rotten Tomatoes ratings integration in Phase 6</decision>
  <context>
Research from 06-RESEARCH.md found these options:

| Option | Free Tier | TMDB ID Lookup | Reliability | RT Score |
|--------|-----------|----------------|-------------|----------|
| MDBList API | 1,000 req/day | By TMDB ID directly | MEDIUM | Yes (Tomatometer + Audience) |
| OMDb API | 1,000 req/day | By IMDB ID (needs mapping) | MEDIUM | Partial (not always present) |
| Official RT API | No public API | N/A | N/A | N/A |
| RapidAPI scrapers | Paid/tiny free | Varies | LOW | Varies |

**Recommendation: MDBList API** — accepts TMDB ID directly (no mapping needed), returns RT Tomatometer + Audience Score, 1,000 free requests/day. Scores can be days-to-weeks stale. Cache in Movie model `rt_score` column.

**OMDb** is second choice but requires adding `imdb_id` column and TMDB-to-IMDB mapping.

**Implementation if approved:** Add `rt_score: int | None` and `rt_audience_score: int | None` to Movie model, fetch from MDBList on demand, cache in DB, display in eligible movies table and movie splash.
  </context>
  <options>
    <option id="implement-mdblist">
      <name>Implement with MDBList API</name>
      <pros>TMDB ID native lookup, returns both RT scores, free tier sufficient for on-demand use with DB caching</pros>
      <cons>Scores can be stale (days-to-weeks), adds external dependency, 1000/day limit requires caching strategy</cons>
    </option>
    <option id="implement-omdb">
      <name>Implement with OMDb API</name>
      <pros>Well-known, more established</pros>
      <cons>Requires IMDB ID mapping (extra column + TMDB fetch), RT score not always present in response</cons>
    </option>
    <option id="defer">
      <name>Drop from Phase 6</name>
      <pros>No additional complexity, no external dependency, ship faster</pros>
      <cons>No RT scores in the app</cons>
    </option>
  </options>
  <resume-signal>Select: implement-mdblist, implement-omdb, or defer</resume-signal>
</task>
