The issue is that client-side filters (genres, MPAA, runtime from MovieFilterSidebar) only apply to the loaded page.

The existing pattern for cross-page filtering: when `filteringByMark` is true (save/shortlist/suggested active), `effectivePage` is set to 1 and `effectivePageSize` to 9999 (line ~153-155). This fetches all results.

Extend this pattern:
1. Change the `filteringByMark` condition (line ~153) to also include when search or sidebar filters are active:
```typescript
const hasActiveFilters = filters.genres.length > 0 || filters.mpaaRatings.length > 0 || filters.runtimeRange[0] !== 0 || filters.runtimeRange[1] !== 300
const filteringByMark = showSavedOnly || showShortlistOnly || showSuggestedOnly
const needsAllResults = filteringByMark || !!debouncedSearch || hasActiveFilters
const effectivePage = needsAllResults ? 1 : moviesPage
const effectivePageSize = needsAllResults ? 9999 : 20
```

2. Update the query key to use `needsAllResults` instead of `filteringByMark` where appropriate (line ~157).
3. Update pagination visibility check (line ~1130): change `!debouncedSearch && !filteringByMark` to `!needsAllResults`.
