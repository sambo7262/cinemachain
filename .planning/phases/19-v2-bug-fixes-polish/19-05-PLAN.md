**Optimistic UI for save/shortlist:**

Currently `saveMovieMutation` and `unsaveMovieMutation` use `onSuccess` to invalidate queries (lines ~374-381). This causes a round-trip delay before the UI updates.

Add `onMutate` for optimistic updates:
```typescript
const saveMovieMutation = useMutation({
  mutationFn: (tmdbId: number) => api.saveMovie(sid, tmdbId),
  onMutate: async (tmdbId) => {
    await queryClient.cancelQueries({ queryKey: ["eligibleMovies", sid] })
    queryClient.setQueryData<PaginatedMoviesDTO>(
      ["eligibleMovies", sid, selectedActor?.tmdb_id ?? null, sortCol, sortDir, allMovies, debouncedSearch, effectivePage, needsAllResults],
      (old) => old ? { ...old, items: old.items.map(m => m.tmdb_id === tmdbId ? { ...m, saved: true } : m) } : old
    )
  },
  onSettled: () => queryClient.invalidateQueries({ queryKey: ["eligibleMovies", sid] }),
})
```

Apply the same optimistic pattern to `unsaveMovieMutation` (set `saved: false`), `addToShortlistMutation` (set `shortlisted: true`), and `removeFromShortlistMutation` (set `shortlisted: false`).

IMPORTANT: Plan 03 renames `filteringByMark` to `needsAllResults` in the useQuery key array. The setQueryData key above uses `needsAllResults` to match. Also note that Plan 03 adds `excludeNR` to the query key — ensure the setQueryData key array matches the actual useQuery key at time of execution. The safest approach is to use `queryClient.setQueriesData` with a partial key match `{ queryKey: ["eligibleMovies", sid] }` to avoid key mismatch issues.

```tsx
<Tooltip key={def.key}>
```
