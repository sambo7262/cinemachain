  <name>Task 1: Create compose.yaml with all Synology constraints</name>
  <files>compose.yaml</files>
  <action>
Create compose.yaml at the project root. Use Compose v2 format — omit the top-level `version:` key (obsolete in Compose v2; causes a warning in DSM Container Manager).

Implement exactly four services in this order: tailscale, backend, postgres, frontend.

Apply ALL Synology constraints from the research (RESEARCH.md Pattern 1). The full service definitions to implement:

**tailscale service:**
- image: tailscale/tailscale:latest
- container_name: cinemachain-tailscale
- hostname: cinemachain
- restart: unless-stopped
- mem_limit: 128m
- environment: TS_AUTHKEY, TS_STATE_DIR=/var/lib/tailscale, TS_USERSPACE=true (use numeric 1, not true — DSM UI compatibility), TS_SERVE_CONFIG=/config/ts-serve.json
- volumes: /volume1/docker/appdata/cinemachain/tailscale:/var/lib/tailscale
- cap_add: NET_ADMIN, NET_RAW

Note: TS_USERSPACE must be set to `1` (not `true`) because DSM Container Manager UI can mangle boolean strings. The Tailscale image accepts both.

**postgres service:**
- image: postgres:16-alpine
- container_name: cinemachain-postgres
- restart: unless-stopped
- mem_limit: 256m
- ports: "127.0.0.1:5433:5432" (loopback bind, not 0.0.0.0)
- environment: POSTGRES_USER=cinema, POSTGRES_PASSWORD=${DB_PASSWORD}, POSTGRES_DB=cinemachain, PGDATA=/var/lib/postgresql/data/pgdata
- volumes: /volume1/docker/appdata/cinemachain/postgres:/var/lib/postgresql/data
- healthcheck: test: ["CMD-SHELL", "pg_isready -U cinema -d cinemachain"], interval: 10s, timeout: 5s, retries: 5, start_period: 30s

**backend service:**
- build: ./backend
- container_name: cinemachain-backend
- restart: unless-stopped
- mem_limit: 512m
- network_mode: service:tailscale
- environment: PUID=${PUID}, PGID=${PGID}, DATABASE_URL=postgresql+asyncpg://cinema:${DB_PASSWORD}@postgres:5432/cinemachain (internal port 5432 — host 5433 is only for external access), TMDB_API_KEY=${TMDB_API_KEY}, PLEX_TOKEN=${PLEX_TOKEN}, PLEX_URL=${PLEX_URL}, RADARR_URL=${RADARR_URL}, RADARR_API_KEY=${RADARR_API_KEY}, SONARR_URL=${SONARR_URL}, SONARR_API_KEY=${SONARR_API_KEY}
- volumes: /volume1/docker/appdata/cinemachain/backend:/app/data
- depends_on: postgres: condition: service_healthy
- command: bash -c "alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port 8000"

**frontend service:**
- build: ./frontend
- container_name: cinemachain-frontend
- restart: unless-stopped
- mem_limit: 64m
- ports: "3000:80"
- depends_on: [backend]
