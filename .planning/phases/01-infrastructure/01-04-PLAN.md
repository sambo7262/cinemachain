<!-- Tailscale Serve config contracts from RESEARCH.md Pattern 3 -->

ts-serve.json structure:
{
  "TCP": {
    "443": { "HTTPS": true }
  },
  "Web": {
    "<hostname>.<tailnet>.ts.net:443": {
      "Handlers": {
        "/": { "Proxy": "http://127.0.0.1:8000" }
      }
    }
  }
}

The hostname in the Web key must match the actual Tailscale machine hostname.
The executor will run: docker exec cinemachain-tailscale tailscale status
to get the full hostname, then write ts-serve.json with that value.

compose.yaml mounts ts-serve.json as:
  TS_SERVE_CONFIG=/config/ts-serve.json
which means the file must be available at that path inside the container.
This requires adding a volume mount to the tailscale service in compose.yaml:
  - ./frontend/ts-serve.json:/config/ts-serve.json:ro
</interfaces>

<task type="auto">
  <name>Task 1: Create Tailscale Serve config and volume directories</name>
  <files>frontend/ts-serve.json, volumes/tailscale/.gitkeep, volumes/postgres/.gitkeep</files>
  <action>
Step 1 — Start Tailscale container alone to discover the tailnet hostname:
```bash
docker compose up -d tailscale
sleep 15
docker exec cinemachain-tailscale tailscale status
```

The output will show the machine hostname in format: `<machine>.<tailnet>.ts.net`.
Extract this full hostname (e.g., `cinemachain.tail1234.ts.net`).

Step 2 — Write frontend/ts-serve.json using the discovered hostname:
```json
{
  "TCP": {
    "443": {
      "HTTPS": true
    }
  },
  "Web": {
    "cinemachain.<your-tailnet>.ts.net:443": {
      "Handlers": {
        "/": {
          "Proxy": "http://127.0.0.1:8000"
        }
      }
    }
  }
}
```
Replace `cinemachain.<your-tailnet>.ts.net` with the actual hostname from `tailscale status`.

Step 3 — Add the ts-serve.json volume mount to compose.yaml tailscale service.
Open compose.yaml, find the `tailscale:` service volumes block, and add:
```yaml
      - ./frontend/ts-serve.json:/config/ts-serve.json:ro
```
This makes the serve config available inside the container at the path TS_SERVE_CONFIG expects.

Step 4 — Create volume directory placeholders:
```bash
mkdir -p volumes/tailscale volumes/postgres
touch volumes/tailscale/.gitkeep volumes/postgres/.gitkeep
```

Note: These are local placeholders only. The actual bind-mounts in compose.yaml point to /volume1/docker/appdata/cinemachain/ on the NAS. The volumes/ directory in the repo is just for documentation/gitkeep purposes. The operator must create the NAS paths via SSH (see user_setup instructions).
