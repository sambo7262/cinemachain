# 04-08 — Cache optimization: actor short-circuit + nightly actor pre-fetch + stub backfill

## Objective

Three targeted backend changes to eliminate on-demand TMDB latency during gameplay:

1. **Short-circuit `_ensure_actor_credits_in_db`** — skip TMDB call if actor already has Credit rows in DB. This fixes the primary slow-load cause: the function currently hits TMDB unconditionally on every eligible-movies request.

2. **Nightly actor pre-fetch** — add a top-1500 popular actors pass to `nightly_cache_job`. Fetches `/person/popular` (75 pages × 20 = 1500 actors) and runs `_ensure_actor_credits_in_db` for each, so mainstream actors are pre-loaded before any user interaction.

3. **Nightly movie stub backfill** — add a pass to `nightly_cache_job` that finds Movie rows with `title=""` or `genres IS NULL` and calls `_ensure_movie_details_in_db` to enrich them. Fixes blank titles/posters in the combined eligible-movies view.

## Tasks

### Task 1 — Short-circuit in `_ensure_actor_credits_in_db` (game.py)

At the top of the function, before any TMDB calls, check if Credit rows already exist for this actor:

```python
# Short-circuit: if credits already in DB, skip TMDB call
existing = await db.execute(
    select(func.count()).select_from(Credit)
    .join(Actor, Actor.id == Credit.actor_id)
    .where(Actor.tmdb_id == actor_tmdb_id)
)
if existing.scalar_one() > 0:
    return
```

Import `func` from sqlalchemy if not already imported.

### Task 2 — Nightly actor pre-fetch (cache.py)

After the existing movie pre-fetch loop completes, add an actor pre-fetch pass:

```python
# --- Actor pre-fetch pass ---
logger.info("nightly_cache_job: fetching top actors for pre-population")
actor_ids: list[int] = []
for actor_page in range(1, 76):  # 75 pages × 20 = 1500 actors
    try:
        r = await tmdb._client.get("/person/popular", params={"page": actor_page})
        r.raise_for_status()
        for person in r.json().get("results", []):
            actor_ids.append(person["id"])
        await asyncio.sleep(0.05)
    except Exception as exc:
        logger.error("nightly_cache_job: actor page %d failed: %s", actor_page, exc)
        break

logger.info("nightly_cache_job: pre-fetching credits for %d actors", len(actor_ids))
async with _bg_session_factory() as db:
    for actor_id in actor_ids:
        await _ensure_actor_credits_in_db(actor_id, tmdb, db)
        await asyncio.sleep(0.05)
logger.info("nightly_cache_job: actor pre-fetch complete")
```

Note: `_ensure_actor_credits_in_db` now short-circuits for already-cached actors, so this loop is fast after first run.

### Task 3 — Movie stub backfill (cache.py)

After the actor pre-fetch, add a stub backfill pass:

```python
# --- Movie stub backfill ---
logger.info("nightly_cache_job: backfilling movie stubs")
async with _bg_session_factory() as db:
    stub_result = await db.execute(
        select(Movie.tmdb_id).where(
            (Movie.title == "") | Movie.genres.is_(None)
        ).limit(2000)
    )
    stub_ids = [row[0] for row in stub_result.all()]
logger.info("nightly_cache_job: %d movie stubs need backfill", len(stub_ids))

if stub_ids:
    async with _bg_session_factory() as db:
        await _ensure_movie_details_in_db(stub_ids, tmdb, db)
logger.info("nightly_cache_job: stub backfill complete")
```
