## Specific Interaction Contracts

### Radarr Notification Banner

- **Mount point:** Rendered inside the root layout wrapper in `App.tsx`, as a sibling immediately after `<NavBar />`, before page `<Routes />`
- **State lift:** `radarrStatus` state lifted from `GameSession.tsx` to React context (`RadarrNotificationContext`) or a lightweight Zustand/atom store — executor's discretion on mechanism
- **Text:** `"Already in Radarr"` or `"Movie Queued for Download"` — exact strings, no variants
- **Auto-dismiss:** 5000ms `setTimeout` on mount clears the message
- **Manual dismiss:** × button (lucide-react `X` icon, 16px) on the right side of the banner
- **When empty:** banner renders as zero-height (no layout shift) — conditionally render nothing when message is null

### Session Actions Menu (`...` button)

- **Trigger:** `...` button (lucide-react `MoreHorizontal` icon) in the session home page header actions area, replacing or alongside the Export CSV button
- **Items in order:**
  1. Export CSV (migrated from current location)
  2. Delete Last Step (destructive — use red text `text-destructive`)
  3. Archive Session (if currently surfaced; conditional on session status)
- **Opens:** `DropdownMenu` from shadcn, aligned to trigger right edge

### Delete Last Step

- **Trigger:** "Delete Last Step" item in session actions menu
- **Confirmation:** `DeleteConfirmDialog` with:
  - Title: `"Delete Last Step"`
  - Body: `"This will remove the most recent step and revert the session to the previous movie. This cannot be undone."`
  - Cancel button: `"Cancel"`
  - Confirm button: `"Delete Step"` with `variant="destructive"`
- **After confirm:** `DELETE /sessions/{id}/steps/last` — session home re-fetches and re-renders to prior state
- **No Radarr cancellation** — download stays in Radarr queue; no mention of this in the UI

### Delete Archived Session

- **Trigger:** "Delete Session" button on the ArchivedSessions page, per session row
- **Confirmation:** `DeleteConfirmDialog` with:
  - Title: `"Delete Session"`
  - Body: `"This will permanently remove this session and all its steps. This cannot be undone."`
  - Cancel button: `"Cancel"`
  - Confirm button: `"Delete Session"` with `variant="destructive"`
- **After confirm:** `DELETE /sessions/{id}` — session removed from archived list; list re-fetches

### Now Playing Hero Poster

- **Dimensions:** `w-[120px] h-[180px]` (2:3 portrait ratio), `rounded-md`, `object-cover`
- **Image URL:** `https://image.tmdb.org/t/p/w185{poster_path}` (w185 size — adequate for 120px display)
- **Fallback:** `<div className="w-[120px] h-[180px] rounded-md bg-muted flex-shrink-0" />`
- **Layout:** poster anchored left inside the Now Playing card flex row; title + status + CTA buttons occupy remaining horizontal space to the right

### MovieCard — MPAA Rating Addition

- **New prop:** `mpaa_rating?: string | null`
- **Display:** `Badge variant="outline"` with MPAA value (e.g., "PG-13", "R", "NC-17"), placed after the genre badges in the existing badge row
- **When null or empty string:** do not render the MPAA badge — no placeholder
- **Always visible:** no hover or expansion required — inline with genre tags

### Suggested Tab

- **Tab label:** `"Suggested"` — third tab after `"Eligible Actors"` and `"Eligible Movies"`
- **Content:** exactly 5 `MovieCard` entries; if fewer than 5 are available, render what is available
- **Attribution:** `via_actor_name` prop passed to each `MovieCard` — displays as `"via [Actor Name]"` (existing italic muted pattern from MovieCard.tsx line 91)
- **Selectable:** yes — clicking a suggested movie goes through the same confirm → request-movie flow
- **Empty state:** if 0 suggestions, show `"No suggestions available yet. Continue the chain to unlock suggestions."`

### Filter Sidebar — Persistent Desktop

- **Desktop:** sidebar is a `<aside>` with `w-[200px] flex-shrink-0`; no collapsible wrapper; always visible
- **Mobile:** sidebar hidden (`hidden lg:block`) on the aside; a `"Filters"` `<Button variant="outline">` appears above the movie grid; clicking opens a `Sheet` or toggles a collapsible above the grid
- Executor discretion: Sheet vs top-collapsible for mobile — Sheet is preferred (shadcn Sheet component, requires `@radix-ui/react-dialog` already needed for confirms)

### Watch History Tab Removal (GameLobby)

- Remove the "Watch History" tab from the session creation tabs in `GameLobby.tsx`
- Retain only: `"Search Title"` and `"Import Chain"` tabs
- Update the default selected tab accordingly
