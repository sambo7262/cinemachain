    Open the NAS app at your Tailscale IP. Perform each verification in order:

    **Test 1 — APScheduler active (CACHE-01)**
    - Check backend container logs via `docker compose logs backend --tail=100`
    - Confirm: log line "APScheduler started" present at startup
    - Confirm: no errors during startup

    **Test 2 — Layout at 1400px (UX-09)**
    - Open app on a wide desktop monitor or browser window
    - Confirm: content area expands and no longer is constrained to a narrow center column
    - Confirm: no horizontal overflow or layout breaks on mobile (resize browser to 375px width)

    **Test 3 — Radarr notification banner (UX-06)**
    - Start a game session (or open existing active session)
    - Select an actor and pick an unwatched movie to trigger a Radarr request
    - Confirm: blue banner appears BELOW the NavBar, spanning full width
    - Confirm: text is exactly "Movie Queued for Download" or "Already in Radarr"
    - Confirm: banner auto-dismisses after ~5 seconds
    - Confirm: × button dismisses immediately when clicked
    - Confirm: banner does NOT overlap session controls or game content

    **Test 4 — Session home poster thumbnail (UX-07)**
    - On session home page, view the Now Playing card
    - Confirm: movie poster appears on the LEFT side of the card, ~120px wide, portrait ratio
    - Confirm: title and CTA buttons appear to the RIGHT of the poster
    - Confirm: if poster_path is missing (old session), grey rectangle appears at same dimensions

    **Test 5 — Watch History tab removed (UX-09)**
    - Navigate to home page (session list / GameLobby)
    - Click "+ Start a new session" or similar creation flow
    - Enter a session name
    - Confirm: only "Search Title" and "Import Chain" tabs are present — no "Watch History" tab

    **Test 6 — Movie card MPAA badge (UX-08)**
    - In an active session, open the Eligible Movies tab (pick any actor)
    - Confirm: movie cards show MPAA rating badge (e.g., "PG-13", "R") alongside genre badges
    - Confirm: MPAA badge visible without hover

    **Test 7 — Persistent filter sidebar (UX-09)**
    - On Eligible Movies tab (desktop browser, wide window)
    - Confirm: filter sidebar is visible on the LEFT side WITHOUT clicking any expand/toggle
    - Confirm: on mobile (or narrow browser), filter sidebar is hidden; "Filters" button is visible above the grid

    **Test 8 — Suggested tab (UX-09)**
    - In an active session with at least 1 step, open tab view and click "Suggested"
    - Confirm: tab exists and content loads
    - Confirm: each suggestion shows "via [Actor Name]" attribution
    - Confirm: if <5 suggestions exist, only available ones are shown (not 5 placeholder cards)
    - Confirm: if 0 suggestions, "No suggestions yet" message is shown

    **Test 9 — Delete Last Step (SESSION-01)**
    - In an active session with ≥2 steps, click the ⋯ button in the session home header
    - Confirm: dropdown opens with "Export CSV" and "Delete Last Step" items
    - Confirm: "Delete Last Step" is red/destructive colored
    - Click "Delete Last Step"
    - Confirm: shadcn Dialog appears (NOT browser alert) with:
      - Title: "Delete Last Step"
      - Body: "This will remove the most recent step and revert the session to the previous movie. This cannot be undone."
      - Cancel: "Keep Step" button
      - Confirm: "Delete Step" (red/destructive) button
    - Click "Keep Step" — confirm dialog closes, session unchanged
    - Re-open dialog and click "Delete Step"
    - Confirm: session home reverts to prior movie; step count decreases by 1
    - Verify: ⋯ menu item "Delete Last Step" is DISABLED when session has only 1 step

    **Test 10 — Delete Archived Session (SESSION-02)**
    - Navigate to Archived Sessions page
    - Confirm: each session row has a "Delete Session" button alongside the "View" button
    - Click "Delete Session" on any archived session
    - Confirm: shadcn Dialog appears with:
      - Title: "Delete Session"
      - Body: "This will permanently remove this session and all its steps. This cannot be undone."
      - Cancel: "Keep Session"
      - Confirm: "Delete Session" (red/destructive)
    - Click "Keep Session" — dialog closes, session remains
    - Re-open dialog and confirm delete
    - Confirm: session disappears from the archived list
