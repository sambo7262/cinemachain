## Phase Character

Phase 4.1 is a backend-dominant bug-fix phase. Three known bugs are resolved:

- **BUG-01:** CSV chain movies appearing as eligible picks (backend exclusion logic)
- **BUG-02:** CSV parsing fails on movie titles containing commas (backend + possible format change)
- **BUG-03:** Suggested movies empty on long chains 130+ movies (backend algorithm)

The frontend impact is minimal: only the CSV file input `accept` attribute and the import instruction label change if `.xlsx` support replaces `.csv`. No new components, pages, or layout regions are introduced. All Phase 4 design tokens, patterns, and interaction contracts carry forward unchanged.

## CSV Format Decision Gate

BUG-02 resolution requires a format decision before implementation:

| Option | Frontend change | User impact |
|--------|----------------|-------------|
| **A — Keep CSV, fix parser** | None — `accept=".csv"` unchanged | No change for existing users |
| **B — Add .xlsx support alongside .csv** | `accept=".csv,.xlsx"` on file input; instruction label updated | Users can upload either format |
| **C — Replace CSV with .xlsx only** | `accept=".xlsx"` on file input; instruction label updated | Existing .csv exports become unusable without conversion |

**Recommendation:** Option B (additive). Existing `.csv` exports from the app's Export CSV feature continue to work. Users with comma-in-title data can export from Excel as `.xlsx` and re-import.

**If Option B or C is chosen**, the only frontend changes are:

1. `GameLobby.tsx` line 376: `accept=".csv"` → `accept=".csv,.xlsx"` (Option B) or `accept=".xlsx"` (Option C)
2. Instruction label (line 364): `"Upload a CSV with columns: order, movie_name, actor_name"` → see Copywriting Contract below

**Executor note:** The backend parser change (xlrd/openpyxl for .xlsx) is backend-only. The frontend only changes the `accept` attribute and the help text.

## Specific Interaction Contracts

### CSV File Input — Format Change (if Option B or C chosen)

- **Component:** The hidden `<input type="file">` in `GameLobby.tsx` (currently at line 374–378)
- **Current:** `accept=".csv"` / trigger button label: `{csvFileName ?? "Choose CSV file"}`
- **Change (Option B):** `accept=".csv,.xlsx"` — no label change required; filename displays on selection
- **Change (Option C):** `accept=".xlsx"` — instruction text updated per Copywriting Contract
- **No other structural change** — FileReader + text parsing is replaced by backend parsing; frontend still reads file bytes and POSTs to the existing `/api/sessions/import-csv` endpoint (or a new endpoint if the backend strategy changes)
- **Error display:** existing `toast("Failed to import CSV chain.")` pattern is retained for parse failures

### Suggested Movies Tab — Long Chain Fix (BUG-03)

- **No UI change required.** The fix is purely algorithmic in the backend suggestions endpoint.
- The `"Suggested"` tab, `"No suggestions yet"` empty state, and all `MovieCard` rendering remain exactly as specified in Phase 4 UI-SPEC.
- If the backend now returns suggestions on long chains where it previously returned empty, the tab will populate correctly with no frontend change.

### CSV Chain Movie Eligibility — Eligible Actors/Movies (BUG-01)

- **No UI change required.** The fix ensures session step movies are excluded from eligible picks at the backend level.
- The Eligible Actors and Eligible Movies tabs render from existing API responses — no new states, no new copy, no new visual treatment for "excluded because in chain" (they simply no longer appear).
