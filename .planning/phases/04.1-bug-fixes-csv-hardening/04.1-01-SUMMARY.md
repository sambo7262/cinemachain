## Decisions Made

- The `if len(top5) < 5 and genre_freq:` fallback condition at line ~1769 required no change — it already handles the `top5 == []` case correctly once early returns removed
- splitCsvLine state-machine approach chosen over regex or PapaParse: zero new dependencies, handles all RFC 4180 cases (quoted commas, escaped double-quotes `""`, CRLF)
- accept=".csv,.xlsx" is UI prep only per UI-SPEC Option B — xlsx backend parsing not implemented; FileReader binary read will fail gracefully with existing error toast
