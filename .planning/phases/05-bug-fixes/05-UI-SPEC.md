## Responsive Layout Contract

This is the primary new visual contract for Phase 5 (BUG-2).

### MovieCard — Portrait Mobile Fix

**Problem:** `CardContent` in `MovieCard.tsx` lacks `min-w-0`, causing flex overflow at narrow widths. Title and metadata truncate in portrait mode.

**Contract:**
- `CardContent` must have `min-w-0` to prevent flex child overflow
- Title (`text-lg font-semibold`) must use `break-words` or `line-clamp-2` to wrap gracefully
- Poster: keep fixed `w-16 h-24 flex-shrink-0` — do not scale or hide the poster
- All five required fields must be visible in portrait at `min-width: 320px`:
  1. Movie title
  2. Via actor name (`text-xs italic text-muted-foreground`)
  3. TMDB rating (star + number, `text-sm text-amber-400`)
  4. Runtime (`text-sm text-muted-foreground`)
  5. MPAA rating (`Badge variant="outline" text-xs`)
- Use Tailwind breakpoint `sm:` (640px) — not CSS orientation media query — for any breakpoint-specific overrides, per existing project convention (`ChainHistory.tsx` `hidden sm:table-cell` pattern)

### GameLobby — Mobile Button Alignment Fix

**Problem:** Session card buttons (Archive, Continue) collapse on mobile. `+ Start a new session` button misaligns with `self-start` at narrow widths.

**Contract:**
- Session action buttons: use `flex-wrap gap-2` on the button group container to allow wrapping on narrow screens
- `+ Start a new session` button: replace `self-start` with `w-full sm:w-auto` so it fills narrow screens and is auto-width on desktop
- No visual style changes — only layout/flex fixes

## New Interaction: BUG-1 Actor Disambiguation Dialog

When `request_movie` returns `status: "disambiguation_required"`, display an inline Dialog (not a full-screen interruption) asking the user to pick which actor they're "following."

**Contract:**
- Component: `Dialog` (already imported in `GameSession.tsx`)
- Pattern: follow existing `randomPickOpen` / `randomPickMovie` dialog pattern in `GameSession.tsx`
- Trigger: detect `status === "disambiguation_required"` in `requestMovieMutation.onSuccess`
- Content:
  - Heading: "Who are you following?" (`DialogTitle`)
  - Body: "Multiple actors connect this pick. Choose which one to record in your chain." (`DialogDescription`)
  - Actor list: one `Button variant="outline"` per candidate actor, full-width, stacked vertically
  - Each button label: actor name only (no extra metadata)
  - On selection: POST the chosen actor as the actor step, then close dialog
  - Cancel: "Skip (leave actor unrecorded)" — `Button variant="ghost"`, bottom of list
- Dialog width: `max-w-sm` — compact, not full-screen
- No destructive styles in this dialog
