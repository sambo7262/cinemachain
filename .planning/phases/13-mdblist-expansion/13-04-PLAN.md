2. **Add backfill UI section to `frontend/src/pages/Settings.tsx`:**

   Place below the MDBList API key input field. Structure:

   **State variables:**
   ```typescript
   const [backfillRunning, setBackfillRunning] = useState(false)
   const [backfillStatus, setBackfillStatus] = useState<{ fetched: number; total: number; calls_used_today: number; daily_limit: number } | null>(null)
   const [showConfirm, setShowConfirm] = useState(false)
   ```

   **Polling effect:**
   ```typescript
   useEffect(() => {
     if (!backfillRunning) return
     const interval = setInterval(async () => {
       const status = await api.mdblist.getBackfillStatus()  // adjust to actual api structure
       setBackfillStatus(status)
       if (!status.running) {
         setBackfillRunning(false)
         clearInterval(interval)
       }
     }, 2000)
     return () => clearInterval(interval)
   }, [backfillRunning])
   ```

   **On mount: check if backfill is already running** (in case user navigates away and back):
   ```typescript
   useEffect(() => {
     api.mdblist.getBackfillStatus().then(status => {
       if (status.running) {
         setBackfillRunning(true)
         setBackfillStatus(status)
       } else {
         // Show current quota even when not running
         setBackfillStatus(status)
       }
     }).catch(() => {})  // silently fail if backend not reachable
   }, [])
   ```

   **UI elements:**

   a) **Refresh Ratings Data button** — disabled when `backfillRunning` is true
      - On click: fetch status first to get estimated total and current quota, then show confirm dialog

   b) **Confirm dialog** (can use shadcn AlertDialog or a simple conditional div):
      - "This will fetch ratings for approximately {total} movies."
      - "Quota: {calls_used_today} of {daily_limit} calls used today."
      - If total > (daily_limit - calls_used_today): show warning "Estimated calls exceed remaining daily quota. Backfill will stop at the limit."
      - Confirm and Cancel buttons

   c) **Progress section** (visible when backfillRunning or backfillStatus has data):
      - Progress bar: `fetched / total` as percentage width
      - Text: "{fetched} of {total} movies updated"
      - Quota counter: "{calls_used_today} / {daily_limit} API calls today"
      - Use Tailwind: `bg-primary` progress bar inside `bg-muted` track, `h-2 rounded-full`

   d) **Completion state**: When backfill finishes (running goes false), show "Done - {fetched} movies updated" message.

   Use existing shadcn components already imported in Settings.tsx (Button, etc.). Add imports as needed.
