## Gaps

### BUG-1: Session titles show "untitled" — movie attribution missing everywhere

**Observed:** After creating a second session (or navigating back to session 1), the title shows as "untitled" and all movie attribution is gone across the app.

**Root cause:** Backend `GameSessionResponse` does not include `current_movie_title` — only `current_movie_tmdb_id` is stored. The 03.1-05 fix removed the `"Movie ${id}"` fallback string without the backend providing the actual title. The session response needs to JOIN with the `Movie` table to return `current_movie_title: str | None`.

**Fix:** In `game.py`, `_build_session_response` must look up `Movie.title` by `current_movie_tmdb_id` and include it in `GameSessionResponse`. Frontend `GameSession.tsx` displays `session.current_movie_title`.

---

### BUG-2: Second session has no "Mark as Watched" button or game actions

**Observed:** Navigating to a second session shows no interactive buttons — cannot mark as watched, no game actions available.

**Root cause:** `GameSession.tsx` still uses `getActiveSession` (returns only one session) rather than fetching the specific session by ID from the URL params. The second session is never loaded into the component state.

**Fix:** Replace `useQuery(["activeSession"], api.getActiveSession)` with `useQuery(["session", sessionId], () => api.getSession(sessionId))` using the `:id` param from React Router. Also ensure `api.getSession(id)` endpoint exists on the backend.

---

### BUG-3: Chain history has duplicate rows with blanks (same order number repeating)

**Observed:** Chain history table shows the same step number twice — one populated row and one blank row. The export CSV has the same duplication issue, plus the first movie (starting movie) is missing.

**Root cause:** `_enrich_steps_watched_at` does a LEFT JOIN between `GameSessionStep` and `WatchEvent` on `movie_tmdb_id`. If there are multiple `WatchEvent` rows for the same movie, the JOIN produces duplicate `GameSessionStep` rows. The starting movie step (where `actor_tmdb_id IS NULL`) may be getting filtered by the enrichment query.

**Fix:** Use a subquery or `DISTINCT ON` to ensure at most one WatchEvent row per `movie_tmdb_id`. Confirm the starting movie step is included (actor_tmdb_id=NULL should not be filtered). Consider using `MAX(watched_at)` grouped by `movie_tmdb_id`.

---

### BUG-4: CSV export missing first movie; duplicate rows

**Observed:** CSV export for a session is missing the first/starting movie row. Other rows repeat with blanks (same order, one blank).

**Root cause:** Same as BUG-3 — the export iterates over enriched steps. The starting movie step (actor_tmdb_id=NULL) is being dropped somewhere in the enrichment or serialization. Duplicate rows from the JOIN carry through to the export.

**Fix:** Resolved by BUG-3 fix. Additionally ensure the starting step (where actor_tmdb_id is NULL) is explicitly included in the export with `actor_name = ""`.

---

### BUG-5: CSV import button missing from UI

**Observed:** No way to import a CSV chain — the import flow was folded into the new-session creation form and is not accessible as a standalone action.

**Fix:** Add a dedicated "Import Chain" card or tab on the home page (session grid), separate from the new-session creation form. This is a standalone action that creates a new named session from a CSV file.

## New Requirements (surfaced during verification)

### R1: Remove pause/resume functionality entirely

**Scope:** Both frontend and backend.
- Remove `pause` and `resume` endpoints from `game.py`
- Remove `SessionStatus.paused` and `SessionStatus.awaiting_continue` if no longer needed (or keep as dead enum values — migration not required if just removing endpoint)
- Remove Pause/Resume buttons from `GameSession.tsx`
- Remove `pauseMutation` and `resumeMutation` from frontend

**Confirmed by user:** Yes, remove completely.

### R2: Remove "End Session" button — archive is the replacement

**Scope:** Frontend only (archive already exists as the lifecycle endpoint).
- Remove the End Session button from `GameSession.tsx`
- Archive button on the home page session card is the replacement
- No backend endpoint removal required (keep `end_session` for potential future use or remove — user preference)

**Confirmed by user:** Yes, remove from UI.

### R3: Home page session cards — clearer "active movie" indicator

**Scope:** Frontend only.
- Session cards on the home page should clearly show the current movie name (not just a text label)
- Should be visually obvious which movie the session is "on" — consider showing the movie poster thumbnail or a styled movie name badge

**Note:** This is partially addressed by BUG-1 fix (current_movie_title must be in the response). Once the title is populated, the card display needs to use it prominently.
