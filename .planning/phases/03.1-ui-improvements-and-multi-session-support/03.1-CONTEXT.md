# Phase 03.1: UI Improvements and Multi-Session Support — Context

**Gathered:** 2026-03-15
**Status:** Ready for planning
**Source:** User conversation — requirements captured directly

<domain>
## Phase Boundary

This phase improves the Movie Game UX in three areas:
1. **Multi-session support** — remove the single-session gate so multiple independent sessions can run concurrently (different users on the network, different game nights)
2. **Session management UI** — naming, archiving, navigation between sessions
3. **In-session UX polish** — chain history table, breadcrumb removal, TMDB ID → movie name display, CSV export/import fixes

This phase does NOT change the core game loop logic (GAME-01 through GAME-08 are already satisfied). It upgrades the surrounding infrastructure and display layer.

</domain>

<decisions>
## Implementation Decisions

### Multi-Session Support
- Remove the backend gate that enforces one active session at a time
- The original gate was built for Plex webhook matching — now that all interactions are manual, there is no risk of invalid session matching
- Multiple sessions can be `active` simultaneously — each is fully independent
- No authentication required — any user on the network can access any session

### Session Naming
- Each session must have a name (required at creation time, prompted in GameLobby)
- Names must be unique among **active sessions only** — archived sessions do not block name reuse
- Uniqueness enforced both client-side (UI validation) and server-side (DB constraint or query check)
- Name is stored on the `GameSession` model

### Home Page — Active Sessions
- Home page displays all active (non-archived) sessions as cards
- Each card shows: session name, current movie, chain length
- Users can click any session card to navigate to that session
- No session → show "Start a new session" prompt as before

### Session Archiving
- Sessions can be archived — removes them from the home page and main navigation
- Archived sessions are NOT deleted; they remain accessible via a dedicated "Archived Sessions" page in the nav
- Archive is a soft status change (e.g. `status: "archived"`)
- Archived sessions are read-only (no game actions)

### Navigation
- Users can freely move between: Home page, any active session, Archived Sessions page
- NavBar updated to support this navigation model
- "Sessions" nav link → Home (shows all active sessions)

### Breadcrumb Removal
- The breadcrumb at the top of the session page is confusing and must be removed entirely
- No replacement needed — the session name + NavBar provide sufficient context

### Chain History Table (Session Home Page)
- Located below the main section on the session home page (below current movie / action buttons)
- Numbered rows in chain order (Step 1, Step 2, Step 3...)
- Each row represents one link: alternating Movie row / Actor row
  - Movie row: step number, movie poster thumbnail, movie name, timestamp (when marked as watched)
  - Actor row: step number, actor profile thumbnail, actor name
- Scrollable; pagination deferred to a later phase (table can grow long)
- Table is display-only — no actions within the history table

### Movie Name Display (TMDB ID Fix)
- All user-facing text must show movie names, never TMDB numeric IDs
- Currently "Movie 23456" appears in breadcrumbs and other areas — root cause: TMDB ID used where name should be
- Fix across all components: session titles, chain display, breadcrumbs, any other surface
- Backend must return `movie_name` (title string) alongside or instead of `movie_tmdb_id` wherever IDs are currently surfaced to the UI

### CSV Export
- Each session has an "Export CSV" button (location: session home page or session header)
- Export format: `order, movie_name, actor_name`
  - One row per chain step
  - Movie rows: order=N, movie_name="Free Guy", actor_name="" (blank)
  - Actor rows: order=N, movie_name="", actor_name="Ryan Reynolds" (blank movie)
  - OR: interleaved pairs — implementation choice
- Export and import formats must match for roundtrip (export → re-import works cleanly)

### CSV Import Fix
- Current import throws errors around timestamps and other unrecognized columns
- Fix: strip all columns except `order`, `movie_name`, `actor_name` — ignore everything else silently
- Only those three columns are required; all others optional/stripped
- **UI validation step after parsing:**
  - Display the parsed chain as a preview list before importing
  - Validate sequence: movies and actors must form a valid chain (movie → actor → movie → actor)
  - Highlight any invalid rows (unrecognized movie name, broken chain sequence)
  - User can delete individual rows from the preview list until a valid sequence is found
  - Import only proceeds when the sequence is valid (or user force-confirms)
- No external movie/actor validation against TMDB required — sequence structure validation only

### Claude's Discretion
- Exact DB migration approach for `name` field and `archived` status
- Whether archived status is a new `SessionStatus` enum value or a separate boolean column
- Pagination implementation details for chain history (deferred, but schema should support it)
- Exact placement of Export CSV button
- Whether CSV uses interleaved rows or separate movie/actor columns

</decisions>

<specifics>
## Specific Requirements

### Chain history table row format (confirmed by user):
- One row per step, numbered
- Alternating: Movie row (poster, name, timestamp) → Actor row (profile photo, name)
- Timestamp = when the movie was marked as watched
- Scrollable; pagination is a future enhancement

### Session name uniqueness scope (confirmed by user):
- Unique among **active sessions only**
- Archived sessions do not block name reuse

### Multiple sessions backend (confirmed by user):
- Remove the `get_active_session` single-session enforcement
- Multiple `active` sessions allowed simultaneously

### TMDB ID display fix scope (confirmed by user):
- "Movie 23456" appears in breadcrumbs and other UI areas
- Fix everywhere: breadcrumbs, session titles, chain display, any text surface

### CSV roundtrip (confirmed by user):
- Export format matches import format: `order, movie_name, actor_name`
- Export + re-import must work cleanly

</specifics>
