# Phase 03.1 Plan 04: GameLobby Multi-Session Home Page Summary

**GameLobby rewritten as a session-grid home page with name-required creation flow and CSV sequence-validation preview using header-name column detection**

## Accomplishments

- GameLobby now defaults to a session grid showing all active sessions as cards (name, current movie, chain length, Archive + Continue buttons)
- New-session inline form with name input that validates in real-time against active session names and gates the movie-selection tabs until valid
- CSV parser replaced: positional split removed, header-name detection added (`includes("movie")`, `includes("actor")`); each parsed row carries an `isValid` flag for sequence checking
- Import Chain button disabled when sequence is invalid or session name is unset
- All three mutations (create, import, archive) updated to use `queryKey: ["activeSessions"]`

## Task Commits

1. **Task 1: Rewrite GameLobby.tsx** - `e4a9c71` (feat)

## Files Created/Modified

- `frontend/src/pages/GameLobby.tsx` - Complete rewrite: session grid, inline new-session form with name input, CSV validation preview UI

## Decisions Made

- `view: 'grid' | 'form'` enum drives home page state — grid is the permanent default, form expands inline (no modal)
- `queryKey: ["activeSessions"]` (plural) introduced for the list query — avoids cache collision with `["session", id]` used in GameSession.tsx
- `isNameValid` gates the tabs section using Tailwind `opacity-50 pointer-events-none` — no per-tab disabled prop needed
- `currentMovieForSession` helper resolves movie title from steps using `current_movie_tmdb_id` match, with fallbacks to first step and `"(untitled)"`
- `ParsedRow.isValid` flag computed in `parseCSV` — sequence validation runs once on parse, re-runs when rows are deleted via the preview UI
