## Decisions Made
- `name` is required (not optional) on `CreateSessionRequest` — prevents unnamed sessions
- Partial unique index uses WHERE clause filtering archived/ended statuses, so a name can be reused after a session ends or is archived
- `_build_session_response` centralizes response construction to avoid duplication across all endpoints that return `GameSessionResponse`
- Archived guard added to `eligible-actors` as well (not just the 4 mutating endpoints specified in plan) — read access also blocked for archived sessions
- All existing tests updated to pass a unique `name` field since the field is required
