<objective>
Replace the "Load More" / accumulation pattern in the Eligible Movies list with standard prev/next
pagination, and add pagination to ChainHistory.

BUG-04: "Load More" causes sort stability problems — each new page fetch can reorder all results.
Fix: remove accumulation state; each page is an independent slice of the full sorted backend list.
Prev/next controls replace the Load More button.

BUG-05: Resolved as a side-effect — with no accumulation, sort/filter changes always show a clean
page 1 result. The reset-to-page-1 effect already exists (lines 133–136); it now simply works.

ChainHistory: sessions can have 100+ movies. Paginate at 20 steps/page. Search resets to page 1.

Note: ChainHistory's search filter at line 20 also uses a `step_order + 1` assumption
(`sorted.find((s) => s.step_order === step.step_order + 1 && ...)`). This is the same latent bug
as BUG-08 in the CSV export. Fixing it is out of scope for BUG-04/05, but should be noted as
deferred — the BUG-08 pattern fix in Plan 08-04 does not cover this frontend occurrence.
