<objective>
Fix two session-specific bugs:

BUG-08 (CSV export 500 error): `actor_step = step_by_order.get(step.step_order + 1)` assumes the
actor step is always exactly one step_order after the movie step. This fails when steps have gaps
(e.g., step_order 1, 3 — step 2 was deleted). Replace with a forward-scan: find the first step
after this movie step in the sorted list where `actor_tmdb_id is not None`.

BUG-07 (Trainspotting session — no eligible movies): The combined-view eligible movies query returns
empty results when the current movie has no credits in the DB. The session is stuck — the user sees
"No eligible movies" but the real cause is missing cast data, not exhausted actors. Fix: when
`eligible_actor_tmdb_ids` is empty in the combined-view path AND the current movie has no credits,
trigger an on-demand cast re-fetch (`_ensure_movie_cast_in_db`) and retry the actor query. This
self-heals corrupted session data on the next eligible-movies load.
</objective>
